    - When `card_color = "002244"` (default-ish navy), card_name / issuer / expiry all render in `#ffffff`
    - When `card_color = "ffff00"` (pale yellow), card_name / issuer / expiry all render in `#0d1117` (legible on yellow in BOTH light and dark theme)
    - When `card_color` is null/undefined, the fallback `#002244` background is used and text resolves to `#ffffff`
    - The DOM no longer contains the literal strings `style="color: #ffffff;"`, `style="color: rgba(255,255,255,0.75);"`, or `style="color: rgba(255,255,255,0.85);"` for the three text rows — those become `:style` bindings

1. Add a module-local helper (above the `defineProps` call, inside `<script setup lang="ts">`):

```ts
function pickTextColor(hex: string): string {
  const c = hex.replace('#', '').toLowerCase()
  const r = parseInt(c.slice(0, 2), 16) / 255
  const g = parseInt(c.slice(2, 4), 16) / 255
  const b = parseInt(c.slice(4, 6), 16) / 255
  const lin = (v: number) => (v <= 0.03928 ? v / 12.92 : Math.pow((v + 0.055) / 1.055, 2.4))
  const L = 0.2126 * lin(r) + 0.7152 * lin(g) + 0.0722 * lin(b)
  return L > 0.5 ? '#0d1117' : '#ffffff'
}
```

2. Replace the existing `tileStyle` computed with the version that also emits `color`. Add a `textColor` computed that the template can reuse:

```ts
const cardBg = computed(() =>
  props.record.card_color ? '#' + props.record.card_color : '#002244'
)
const cardTextColor = computed(() => pickTextColor(cardBg.value))

const tileStyle = computed(() => ({
  backgroundColor: cardBg.value,
  color: cardTextColor.value,
}))
```

   - `style="color: rgba(255,255,255,0.85);"` (expiry) → replace with `:style="{ color: cardTextColor, opacity: 0.85 }"`

Keep the rest of the template (Badges, the `flex items-center gap-2 mt-auto` row, `min-height: 8rem`, `cursor-pointer`, etc.) exactly as-is.

Per D-05: do NOT modify `props.record.card_color` itself — only the foreground text color derived from it.

Rationale for keeping the helper inline (Claude's Discretion in CONTEXT.md): only two files in scope use `card_color` and only one renders it as a background. Promoting to `src/utils/contrast.ts` adds an import for no shared benefit yet.
