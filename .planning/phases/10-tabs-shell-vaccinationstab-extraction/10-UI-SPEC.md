## Component Structure Contract

### WallecxApp.vue — After Phase 10

WallecxApp.vue becomes a thin shell. It retains the outer `<Card>` wrapper (preserves existing visual boundary and consistent padding). Inside the Card:

```
<Card>
  <template #content>
    <!-- App heading above tabs -->
    <h1 class="text-2xl font-bold mb-4" style="color: var(--color-typo-heading)">
      Wallecx
    </h1>

    <!-- PrimeVue 4 Tabs shell -->
    <Tabs v-model:value="activeTab">
      <TabList>
        <Tab value="vaccinations">
          <iconify-icon icon="mdi:needle" ... /> Vaccinations
        </Tab>
        <Tab value="memberships">
          <iconify-icon icon="mdi:card-account-details-outline" ... /> Membership Cards
        </Tab>
      </TabList>
      <TabPanels>
        <TabPanel value="vaccinations">
          <VaccinationsTab />
        </TabPanel>
        <TabPanel value="memberships">
          <MembershipsTab />
        </TabPanel>
      </TabPanels>
    </Tabs>

    <!-- Single ConfirmDialog instance stays here (app-level) -->
    <ConfirmDialog />
  </template>
</Card>
```

**Rationale for heading above tabs (not inside a tab):** The "Wallecx" heading identifies the entire mini-app, not a specific tab's content. Placing it above the `<TabList>` matches how ProjectsView.vue structures page-level headings and avoids repeating it in both tabs.

**Rationale for keeping the Card wrapper:** Removes one layer of indentation change from the current structure. VaccinationsTab.vue absorbs the existing content verbatim without touching the outermost container; reviewers can diff the extraction cleanly.

### PrimeVue 4 Tabs API

Use the **PrimeVue 4** `<Tabs>` API — NOT the legacy `<TabView>` used in LexTrackApp.vue (which predates PrimeVue 4). The correct components are:

| Component | Role |
|-----------|------|
| `<Tabs v-model:value="activeTab">` | Shell; `activeTab` is a `ref<string>` initialized to `'vaccinations'` |
| `<TabList>` | Renders the clickable tab headers |
| `<Tab value="vaccinations">` | Individual tab header — use string values, not numeric |
| `<TabPanels>` | Panel container |
| `<TabPanel value="vaccinations">` | Panel body — only renders when tab is active (lazy by default in Aura) |

All six components are PrimeVue auto-resolved — no explicit imports required.

### Tab Icons

| Tab | Icon | Library | Rationale |
|-----|------|---------|-----------|
| Vaccinations | `mdi:needle` | iconify-icon | Matches existing `mdi:needle-off` used in the zero-records empty state |
| Membership Cards | `mdi:card-account-details-outline` | iconify-icon | Semantic: wallet/ID card icon; outline weight matches `mdi:needle` style |

Icon size inside tab labels: `width="16" height="16"` (`16px`). Inline with label text, gap `4px` (Tailwind `gap-1`). Use `aria-hidden="true"` on all decorative icons.

### Tab Styling

Use Aura preset defaults — no custom `pt` overrides on the Tabs component. The Aura preset renders:
- Inactive tab label: `--color-typo-muted` equivalent text
- Active tab label: primary color (`#002244` light / `#e89820` dark)
- Active indicator: 2px underline using primary color
- Tab list border-bottom: `--color-surface-divider` equivalent

No custom CSS is needed. The Aura navy primary scale defined in `main.ts` already resolves correctly.

### VaccinationsTab.vue — Content Contract

VaccinationsTab.vue is a **direct extraction** of everything currently inside `<Card><template #content>` in WallecxApp.vue, minus the outer heading. Specifically it contains:

1. All vaccination `ref` state (`records`, `isLoading`, `selectedRecord`, `showDetail`, `fileToken`, `listToken`, `showManage`, `manageRecord`, `isExporting`, `showGroupPanel`, `selectedGroup`, `searchQuery`, `sortMode`, `viewMode`)
2. The `VaccineGroup` interface and `groupedVaccinations` / `displayedGroups` / `gridClass` computed properties
3. All lifecycle hooks (`onMounted`, `onUnmounted`) and watchers
4. All handler functions (`openDetail`, `openManage`, `openGroupPanel`, `exportJson`, `onCreated`, `onUpdated`, `openDelete`, `deleteRecord`)
5. Template: header row (Download + Add buttons), `<WallecxToolbar>`, loading skeleton, zero-records empty state, no-results empty state, grouped card grid, `<Drawer>`, `<ManageVaccination>`, `<Dialog>` (VaccinationDetail)

The `<ConfirmDialog>` component moves to WallecxApp.vue (app-level) because `useConfirm` requires a single `<ConfirmDialog>` instance per app layer — it must not be duplicated.

**Zero visual regression requirement (XTAB-02):** The extracted template is pixel-identical to the current WallecxApp.vue content. No layout, spacing, or color changes are made during extraction.

## Interaction Contract

### Tab switching

- Default active tab: `'vaccinations'` (string value, not numeric index)
- Tab switch is immediate — no animation delay beyond Aura's built-in transition
- Both tabs are always rendered in the DOM when their panel is active; panels are lazy-mounted by default in PrimeVue 4 `<Tabs>` (MembershipsTab.vue is not mounted until the user clicks the tab)
- No URL hash or router changes on tab switch (PrimeVue Tabs, not sub-routes — locked decision in STATE.md)

### VaccinationsTab interaction

All interactions are unchanged from current WallecxApp.vue:
- Search input: real-time filter via `displayedGroups` computed (no debounce)
- Sort select: immediate re-order
- View toggle: immediate grid/list switch, persists to `sessionStorage`
- Group card click: opens `<Drawer>` from right, 30rem wide (`92vw` below 641px)
- Detail row click in drawer: opens `<Dialog>` modal
- Edit button in drawer: opens `<ManageVaccination>` dialog pre-filled
- Delete button in drawer: triggers `useConfirm` → `<ConfirmDialog>` flow

### MembershipsTab interaction

No interactions in Phase 10 — stub is static display only. No click handlers, no dialogs, no data fetching.
