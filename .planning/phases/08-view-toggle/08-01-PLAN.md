CRITICAL implementation notes (from 08-UI-SPEC.md Constraint Checklist):
- `v-if="showToggle"` — Button is omitted entirely from DOM when `showToggle` is false (D-03).
- `severity="secondary"` (NOT `primary`, NOT default) — keeps the toggle neutral against the sort Select neighbor (the brand navy is reserved for the "Add vaccination" CTA).
- `size="small"` — matches the inline "Clear search" Button precedent from Phase 7.
- NO explicit width class (no `class="w-10"`, no `class="w-..."` of any kind). PrimeVue Aura at `size="small"` icon-only renders ~32px square intrinsically.
- The icon expression `viewMode === 'grid' ? 'mdi:view-list' : 'mdi:view-grid'` follows the "icon = destination" rule from D-01: in grid mode show the list icon (meaning "switch to list"), and vice versa.
- The click expression `emit('update:viewMode', viewMode === 'grid' ? 'list' : 'grid')` emits the OPPOSITE of the current mode — the cycle direction.
- `aria-label` and `title` use the IDENTICAL ternary expression — both bindings produce the same string. The native HTML `title` is used (NOT PrimeVue `v-tooltip`) — zero deps, cross-platform.
- The inner `<iconify-icon>` carries `aria-hidden="true"` so the Button's `aria-label` is the only accessible announcement.
- The icon is a Vue custom element (`<iconify-icon>`) — already registered in vite.config.ts as a known custom element (used by existing empty-state icons like `mdi:needle-off`). NO manual import needed.
- NO manual import for `Button` — auto-imported by `unplugin-vue-components` + `PrimeVueResolver`. Adding `import Button from 'primevue/button'` is a constraint-checklist violation.

| T-08-01 | Tampering | WallecxToolbar.vue — `viewMode` prop value used in the icon and aria/title ternaries | accept | The prop is typed as `'grid' \| 'list'` in TypeScript. The ternary expressions are exhaustive binary checks — any non-`'grid'` value falls through to the `'list'` branch. The Vue template never echoes `viewMode` as raw text; it is only used to select between two hardcoded string literals (`'mdi:view-list'` / `'mdi:view-grid'`, and the two aria/title strings). No XSS surface. |
| T-08-02 | Spoofing / aria mismatch | Button aria-label vs visible icon | mitigate | `aria-label`, `title`, and the `<iconify-icon>` `:icon` binding all use the same `viewMode === 'grid' ? ... : ...` ternary form. They cannot drift out of sync because all three are derived from the same prop value at the same render. Acceptance criteria explicitly grep-checks that the identical aria/title ternary string appears at least twice. |
| T-08-03 | Injection (XSS) | New Button + iconify-icon — no user-supplied text rendered | accept | All three dynamic bindings (`:icon`, `:aria-label`, `:title`) select between hardcoded string literals; none echo any user input. | T-08-04 | Information Disclosure | WallecxToolbar.vue — no network or storage I/O | accept | Component is purely presentational. No PocketBase calls, no sessionStorage/localStorage I/O (storage lives in WallecxApp.vue, Plan 2). Zero disclosure surface. |
