```typescript
async function loadBudgets(opts: { context: 'mount' | 'refresh' } = { context: 'refresh' }): Promise<void> {
  try {
    budgets.value = await pb
      .collection('wallecx_expense_budgets')
      .getFullList<ExpenseBudget>({
        requestKey: 'expense-budgets-getFullList',
      })
  } catch (e: unknown) {
    const msg = opts.context === 'mount'
      ? 'Failed to load budgets.'
      : 'Failed to refresh budgets after save. Reload to see changes.'
    toast.error(msg)
    console.error('ExpensesTab: loadBudgets failed', e)
  }
}

onMounted(async () => {
  isLoading.value = true
  try {
    expenses.value = await pb
      .collection('wallecx_expenses')
      .getFullList<Expenses>({
        sort: '-expense_date,-created',
        requestKey: 'expenses-getFullList',
      })
  } catch (e: unknown) {
    toast.error('Failed to load expenses. Pull to refresh or reload the page.')
    console.error('ExpensesTab: getFullList failed', e)
  } finally {
    isLoading.value = false
  }
  await loadBudgets({ context: 'mount' })
})
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → PocketBase API | The two `pb.collection(...).getFullList(...)` calls cross this boundary. Server-side per-user isolation (STATE.md invariant) is the auth control; this phase does not change that posture. |
| PocketBase response → Vue reactive state | Untrusted-shape data lands in `expenses.value` and `budgets.value`. Type assertions (`<Expenses>`, `<ExpenseBudget>`) are compile-time only. |
| Error object → toast UI | `e: unknown` flows into `console.error`; only static literal strings (not `e.message`) reach the user via `toast.error`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-32-01 | Information Disclosure | `toast.error(...)` calls in `loadBudgets` and `onMounted` expenses catch | mitigate | All four toast strings are static string literals — no `e.message`, no record fields, no user-controlled input interpolated into the UI. The ternary in `loadBudgets` selects between two pre-defined literals based on the locally-bound `opts.context` (never user input). |
| T-32-02 | Information Disclosure | `console.error('ExpensesTab: ... failed', e)` | accept | The error object is logged to the browser console only. No PII leaves the client. Existing project pattern; debugging signal is necessary. Matches Phase 26/28 precedent. |
