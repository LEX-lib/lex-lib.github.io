## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline text spacing |
| sm | 8px | Button internal gaps, tight list items |
| md | 16px | Default element and section padding |
| lg | 24px | Section separators, card internal padding |
| xl | 32px | Major layout spacing |
| 2xl | 48px | Large section breaks |
| 3xl | 64px | Page-level vertical rhythm |

Exceptions:
- **44px min touch target** — all interactive elements (buttons, icon buttons, dismiss controls, Install CTA) must meet 44×44px. Enforced by existing `.wallecx-root .p-button.p-button-icon-only` rule in `wallecx-overrides.css`. Phase 37 banner buttons must explicitly set `min-width: 44px; min-height: 44px` (source: LT-01 / 37-CONTEXT.md D-37-03).
- **Safe-area insets** — `paddingBottom: calc(env(safe-area-inset-bottom) + 0.75rem)` on the bottom-fixed install banner (existing iOS branch, reused verbatim for Android branch per D-37-03). `paddingTop: env(safe-area-inset-top)` on the top-fixed offline banner (D-37-10).
- **52px sticky offset** — `.wallecx-tab-toolbar` already uses `calc(env(safe-area-inset-top) + 52px)`. Unchanged in Phase 37.

> Source: wallecx-overrides.css (Phases 34–35); 37-CONTEXT.md D-37-03, D-37-10

## Typography

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.5 |
| Label / Caption | 14px | 400 (regular) | 1.5 |
| Heading | 24px (text-2xl) | 700 (bold) | 1.2 |
| Display | 28px | 700 (bold) | 1.2 |

Notes:
- **Exactly 2 weights declared:** `400 (regular)` and `700 (bold)`. Size differentiates Display (28px) from Heading (24px) — no need for a distinct semibold weight.
- **Mobile input override (FD-01):** All `.p-inputtext`, `.p-inputnumber-input`, `.p-textarea`, `.p-select-label`, `.p-multiselect-label`, `.p-datepicker-input` are forced to `font-size: 16px !important` at `max-width: 640px` — existing rule in `wallecx-overrides.css`. Phase 37 does not add new form inputs, so no new instances of this rule are required.
- Banner copy (`PwaInstallBanner.vue`, `OfflineBanner.vue`) uses `text-sm` (14px) with `color: #ffffff` on navy background. This is NOT a form input — FD-01 16px rule does not apply.

> Source: base.css (`--color-typo-*`), wallecx-overrides.css FD-01, WallecxApp.vue template (text-2xl font-bold heading)

## Interaction Contracts

### Install Banner (PwaInstallBanner.vue)

**Show logic:**
- iOS branch: `isIosSafari() && !isInStandaloneMode() && installPromptEvent === null && !isDismissed()`
- Android branch: `installPromptEvent !== null && !isInStandaloneMode() && !isDismissed()`
- `isStandalone === true` always suppresses (D-37-08)
- localStorage failure silently suppresses (existing behavior)

**Dismissal logic (30-day, D-37-06):**
- Key: `wallecx_pwa_banner_dismissed` (unchanged)
- New value: `JSON.stringify({ dismissedAt: <ISO8601>, platform: 'ios' | 'android' })`
- Re-show gate: `Date.now() - Date.parse(parsed.dismissedAt) >= 30 * 86_400_000`
- Legacy `'true'` migrates to `{ dismissedAt: <now-ISO>, platform: <detected> }` on first read (D-37-07)

**Android Install button interaction (D-37-04):**
1. User taps "Install Wallecx"
2. `installPromptEvent.prompt()` called (FIRST and ONLY call — M-9)
3. `await event.userChoice` resolves
4. On `accepted`: clear singleton via `clearInstallPromptEvent()` — Chrome fires `appinstalled` which also clears it (App.vue teardown)
5. On `dismissed`: write 30-day dismissal record AND clear singleton — banner hides for this session
6. No toast on either outcome (Chrome's own dialog is the feedback)

### Offline Banner (OfflineBanner.vue)

- `useOnline()` ref drives `v-if="!isOnline"`
- No retry button (D-37-11) — `useOnline` reactivity IS the retry mechanism
- Banner disappears automatically when `navigator.onLine` fires `online`
- Teleport to body; z-index 50

### Manifest Shortcuts (PWA-09, D-37-13 / D-37-14)

4 Quick Actions (long-press home-screen icon on Android):

| Shortcut Name | URL | Icon |
|---------------|-----|------|
| Add Expense | `/projects/wallecx?action=add-expense` | `public/shortcuts/shortcut-add-expense.png` (96×96) |
| Add Vaccination | `/projects/wallecx?action=add-vaccination` | `public/shortcuts/shortcut-add-vaccination.png` (96×96) |
| Add Membership | `/projects/wallecx?action=add-membership` | `public/shortcuts/shortcut-add-membership.png` (96×96) |
| Open Reports | `/projects/wallecx?action=open-reports` | `public/shortcuts/shortcut-open-reports.png` (96×96) |

WallecxApp dispatch on mount (D-37-14):
1. Read `route.query.action`
2. Map to `activeTab` string (expenses / vaccinations / memberships / expenses) + set `pendingAction` ref
3. Tab component watches `pendingAction` and opens the appropriate dialog in create mode
4. `router.replace({ query: {} })` clears param after dispatch

Auth fallback: existing `router.beforeEach` preserves `to.fullPath` (includes `?action=…`) in redirect query; after login, WallecxApp re-dispatches naturally.
