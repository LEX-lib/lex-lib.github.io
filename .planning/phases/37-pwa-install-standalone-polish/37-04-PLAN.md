<objective>
Wire the iOS standalone meta tags (PWA-01) and the 3 splash-screen `<link rel="apple-touch-startup-image">` entries (PWA-02 / NFR-IOS-SPLASH) into `index.html`. Plan 01 already generated the splash PNGs at `public/splash/`. This plan attaches them to iOS Safari's install pipeline so a freshly installed PWA shows a branded splash (NOT a white flash) and the status bar renders `black-translucent` with title "Wallecx", and the theme-color flips between navy and dark-surface based on system color scheme.

Purpose: Make the installed iOS experience feel native. Currently `index.html` has zero PWA-related meta tags beyond the LOCKED `viewport-fit=cover`. Phase 14's manifest provides `theme_color: '#002244'` for Android but iOS does NOT read manifest theme_color until iOS 16.4+ for installed PWAs and even then prefers `<meta name="theme-color">`. Per-color-scheme `media` attribute is required for dark-mode status-bar matching.
Output: Single modified `index.html`.
</objective>
