<objective>
Normalize all three existing PocketBase mappers (meeting, support, task) to expose the full mapper triple per D-09 / D-12:
- `mapToCreate<Entity>(input: AddDsu<Entity>) → CreatePayload`
- `mapToUpdate<Entity>(record: Dsu<Entity>) → UpdatePayload` (already exists; preserved)
- `mapFromRecord<Entity>(record: Dsu<Entity> | RecordModel) → Dsu<Entity>` (new — normalizes raw PB output)

Schema-driven additions:
- `mapToUpdateMeeting` and `mapToCreateMeeting` carry `duration_unit` (per D-13/D-14)
- `mapFromRecordMeeting` backfills `duration_unit ?? 'minutes'` for legacy rows lacking the field (per D-10)
- `mapToUpdateSupport` and `mapToCreateSupport` carry `link`
- `mapFromRecordSupport` passes `link` through unchanged (PB's URL field already returns string-or-empty)
- Tasks: pure symmetry (no schema change — same fields as before)

Purpose: Phase 3 will switch `LexTrackView.vue:save()` from passing raw `AddDsu*` to `pb.collection().create()` to using `mapToCreate*` instead. Phase 5 (day-status fetch + render) will adopt the same pattern. Establishing the triple now means Phase 3 doesn't need to refactor the mapper layer — it just calls into the new functions.

Output: Three rewritten mapper files. All existing exports (`mapToUpdateMeeting`, `mapToUpdateSupport`, `mapToUpdateTask`) remain so `LexTrackView.vue:save()` continues to compile unchanged this phase.
</objective>
