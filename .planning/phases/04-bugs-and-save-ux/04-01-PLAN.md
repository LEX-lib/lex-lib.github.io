## Trust Boundaries

| Boundary | Description |

| Dialog component → Parent view | Emit-only contract; parent owns all PB calls (no fetch from this component). User-typed strings (`title`, `description`) flow upward via `defineModel`. |
| User input → DOM render | Vue's template renderer escapes by default; `Editor` v-model on `description` uses Quill's sanitization. No HTML injection vector here. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01-01 | Tampering | `onSaveClick` rapid double-click | accept | PrimeVue `:loading` auto-disables the Save button while saving (RESEARCH.md Pattern 4 / Pitfall #2 acceptable for v1). User cannot fire a second `save` emit while the first is in flight because the parent toggles `props.saving=true` before the next event loop. |
| T-04-01-02 | Information Disclosure | Toast text leaking PB error details | mitigate | Component fires NO toasts. All toast strings are owned by the parent (Plan 04-04) and use D-13 friendly text. This dialog never sees the raw error. |
| T-04-01-03 | Spoofing/Repudiation | Stale auth token (401 on save) | accept (transferred to parent) | Parent's `handleMeetingSave` catch block (Plan 04-04) is the 401 sentinel point per RESEARCH.md "Threat Surface Delta" row. This dialog only emits the event. |
| T-04-01-04 | Injection (XSS) | Toast text constructed from user input would render as HTML | accept | Vue-sonner default `toast(message)` renders text, not HTML [VERIFIED: vue-sonner docs, RESEARCH.md Security Domain]. Plus, this dialog fires no toasts post-refactor. |
