## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Dialog component → Parent view | Emit-only contract; parent owns all PB calls. User-typed strings (`title`, `jira_link`, `description`) flow upward via `defineModel`. |
| `jira_link` input → DOM render | `<InputText type="url">` provides browser-level URL hint; PB validates server-side per Phase 1 D-12. No client-side regex (intentional). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-02-01 | Tampering | `onSaveClick` rapid double-click | accept | PrimeVue `:loading` auto-disables the Save button while parent's save is in flight (RESEARCH.md Pattern 4). |
| T-04-02-02 | Information Disclosure | Toast text leaking PB error details | mitigate | Component fires NO toasts post-refactor. All toast strings owned by parent (Plan 04-04) using D-13 friendly text. |
| T-04-02-03 | Spoofing/Repudiation | Stale auth token (401 on save) | accept (transferred to parent) | Parent's `handleTaskSave` catch block (Plan 04-04) is the 401 sentinel point. This dialog only emits. |
| T-04-02-04 | Injection (XSS) | `jira_link` rendered as anchor or run as URL | accept | This dialog only collects the value into v-model. ActivityCard renders the link via `window.open(url, '_blank', 'noopener,noreferrer')` per Phase 3 D-02 / 03-02-SUMMARY (T-3-01 mitigation already in place). |
</threat_model>
