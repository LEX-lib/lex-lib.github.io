## Layout Contract

### Header Row Pattern (both MembershipsTab and ExpensesTab)

Phase 27 modifies the header row of two components. The modification is identical in structure but differs by tab context.

**MembershipsTab — current structure:**
```
<div class="flex items-center justify-between mb-4">
  <Button label="Add card" ... />   ← sole child, aligned left by default
</div>
```

**MembershipsTab — after Phase 27:**
```
<div class="flex items-center justify-between mb-4">
  <div class="flex gap-2">
    <Button label="Add card" icon="pi pi-plus" size="small" @click="openManage(null)" />
    <Button label="Download records" icon="pi pi-download" severity="secondary" size="small" :disabled="isExporting" :loading="isExporting" @click="exportJson" />
  </div>
</div>
```

> Wrapping both buttons in `<div class="flex gap-2">` keeps them left-aligned together. `justify-between` has nothing to push right (there is no right-side element in MembershipsTab header).

**ExpensesTab — current structure:**
```
<div class="flex items-center justify-between mb-4">
  <h2 class="text-lg font-semibold" style="color: var(--color-typo-heading)">Expenses</h2>
  <Button label="Add Expense" ... />   ← right side via justify-between
</div>
```

**ExpensesTab — after Phase 27:**
```
<div class="flex items-center justify-between mb-4">
  <h2 class="text-lg font-semibold" style="color: var(--color-typo-heading)">Expenses</h2>
  <div class="flex gap-2">
    <Button label="Add Expense" icon="pi pi-plus" size="small" @click="openManage(null)" />
```

> `<h2>` stays on the left. The `<div class="flex gap-2">` group takes the right slot via `justify-between`. "Add Expense" remains the leftmost (primary) button in the group; "Download records" is to its right. This matches Pitfall 5 avoidance from RESEARCH.md.

## Interaction States

### "Download records" Button

| State | Visual | Interaction |
|-------|--------|-------------|
| Default | Secondary severity (gray outline / neutral) — PrimeVue Aura theme default for `severity="secondary"` | Clickable |
| Loading | PrimeVue Button spinner replaces icon (`:loading="isExporting"`) + button is disabled | Not clickable (`:disabled="isExporting"`) |
| Disabled | Same as Loading (the two are always in sync via `isExporting`) | Not clickable |

The `isExporting` ref gates both the `disabled` and `loading` props simultaneously — consistent with the VaccinationsTab POLISH-03 pattern (locked D-10).

### Export Flow State Machine

```
User clicks "Download records"
  → isExporting = true (button enters loading state immediately)
  → pb.authStore null guard check
      → if no session: toast.error("Session expired. Please log in again."); isExporting = false; return
  → getFullList call (requestKey: 'memberships-export' or 'expenses-export')
  → on success: Blob download + toast.success(...)
  → on error: toast.error("Export failed. Please try again.")
  → finally: isExporting = false (button returns to default state)
```

## Accessibility Contract

- "Download records" button: PrimeVue Button renders a native `<button>` element — inherently keyboard-focusable and screen-reader-accessible.
- Loading state: PrimeVue `:loading="isExporting"` renders a spinner and sets `aria-disabled` internally — no manual ARIA annotation needed.
- Toast notifications: `vue-sonner` handles ARIA live region announcements for success/error toasts — no additional work needed.

## Dark Mode Contract

Phase 27 adds no new CSS. All color tokens (`--color-typo-heading`, `--color-surface-card`, etc.) already have `.my-app-dark` overrides in `src/assets/base.css`.
