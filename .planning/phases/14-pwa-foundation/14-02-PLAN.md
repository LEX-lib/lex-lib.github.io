<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Build tool → Generated SW | vite-plugin-pwa generates sw.js at build time; the Workbox config determines what is cached and what is NetworkOnly |
| Browser SW → PocketBase API | Cross-origin PocketBase requests bypass the SW entirely (no interception); same-origin /api/* routes are forced NetworkOnly |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-02-01 | Information Disclosure | SW runtimeCaching — PocketBase responses | mitigate | `handler: "NetworkOnly"` for `/api/.*` in workbox.runtimeCaching — prevents any stale cached PocketBase auth or record data from being served. PocketBase is also cross-origin so SW does not intercept at all (belt-and-suspenders). |
| T-14-02-02 | Denial of Service | SW autoUpdate → unsaved data loss | mitigate | `registerType: "prompt"` is LOCKED — never `"autoUpdate"`. The generated sw.js will NOT contain `skipWaiting: true`. Verified by: `grep "registerType" vite.config.ts` shows `"prompt"`. |
| T-14-02-03 | Information Disclosure | SW serving cached sensitive screens | accept | All PocketBase data routes are cross-origin and NOT intercepted by the SW. Static HTML/JS/CSS precache contains no user data. Risk accepted. |
| T-14-02-04 | Spoofing | Stale manifest cached at CDN | mitigate | `vercel.json` (Plan 01 Task 2) sets `Cache-Control: public, max-age=0, must-revalidate` on `*.webmanifest` — CDN cannot serve stale manifest. |
</threat_model>
