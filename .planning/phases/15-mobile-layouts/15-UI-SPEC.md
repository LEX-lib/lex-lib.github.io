## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline icon-text gap in banner |
| sm | 8px | Compact element spacing, badge margins |
| md | 16px | Default element spacing, banner horizontal padding |
| lg | 24px | Section padding, dialog internal padding |
| xl | 32px | Layout gaps between sections |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- **Touch targets**: minimum 44px height and 44px width on all interactive elements (Apple HIG). Not a spacing token — enforced via `min-h-[44px] min-w-[44px]` per element. Source: CONTEXT.md D-12, REQUIREMENTS.md MOB-02.
- **Safe-area insets**: `env(safe-area-inset-*)` values are device-native and non-fixed. Applied as inline style — not a spacing token.
- **Banner bottom padding**: `calc(env(safe-area-inset-bottom) + 12px)` — 12px (0.75rem) base plus the device inset.

## Interaction Contracts

### PwaInstallBanner.vue — iOS Install Strip

**Visibility rule**: Shown when ALL three conditions are true simultaneously:
1. `navigator.userAgent` matches `/iPad|iPhone|iPod/` AND does NOT match `/CriOS|FxiOS|OPiOS/`
2. `window.matchMedia('(display-mode: standalone)').matches` is `false` AND `window.navigator.standalone !== true`
3. `localStorage.getItem('wallecx_pwa_banner_dismissed') !== 'true'`

**localStorage key**: `wallecx_pwa_banner_dismissed` — stores string `"true"` on dismiss.

**Dismiss**: X button tap only. Sets localStorage key. Sets `isVisible.value = false`. No auto-dismiss timer. No expiry. Permanent.

**Z-index**: `z-50` (Tailwind, z-index: 50). Below PrimeVue Dialogs (~z-index 1100) and below scan overlay (z-index 9999).

**Position**: `position: fixed; bottom: 0; left: 0; right: 0` — full width, anchored to bottom edge.

**Rendering host**: `WallecxApp.vue` template. Teleported to `body` to avoid PrimeVue stacking context issues.

**Error handling**: All `localStorage` access in `try/catch` — private browsing mode may throw. Silent degradation (banner simply does not show if localStorage is unavailable).

### Dialog Scroll (MOB-05)

**CSS rule** (non-scoped, in WallecxApp.vue `<style>` block):
```css
.p-dialog-content {
  max-height: 80dvh;
  overflow-y: auto;
}
```

Applies to: `ManageVaccination.vue`, `ManageMembership.vue`, `VaccinationDetail.vue`, `MembershipDetail.vue` — all four dialog instances, no template changes needed.

**Must NOT use `scoped`** on this style block. Scoped styles add `[data-v-*]` attribute selectors that do not match PrimeVue's internally rendered DOM nodes.

### Safe-area Insets (MOB-06)

**Viewport meta** (after edit):
```
width=device-width, initial-scale=1.0, viewport-fit=cover, interactive-widget=resizes-content
```

**App.vue** — top safe-area on CustomNavBar:
```vue
<CustomNavBar class="mb-1" :style="{ paddingTop: 'env(safe-area-inset-top)' }" />
```

**WallecxApp.vue** — bottom and side safe-areas on `<Card>` root:
```vue
<Card
  class="overscroll-none"
  :style="{
    paddingBottom: 'env(safe-area-inset-bottom)',
    paddingLeft: 'env(safe-area-inset-left)',
    paddingRight: 'env(safe-area-inset-right)',
  }"
>
```

All `env()` values resolve to `0px` on devices without notches — safe to apply unconditionally.

### Touch Targets (MOB-02)

**Minimum**: 44×44px on all interactive elements.

**Tailwind classes applied per element**:
- Buttons (PrimeVue Button, native `<button>`): `min-h-[44px] min-w-[44px] touch-manipulation`
- Card tiles (MembershipCard.vue, VaccinationGroupCard.vue root): `min-h-[44px] touch-manipulation`
- Search-clear icon in WallecxToolbar.vue: requires a `<div class="min-h-[44px] min-w-[44px] flex items-center justify-center touch-manipulation">` wrapper (the `InputIcon` is not a `<button>`)

`touch-manipulation` disables double-tap-to-zoom (removes 300ms iOS tap delay) via `touch-action: manipulation`.

### Responsive Grid (MOB-03, MOB-04)

**Grid class** (already correct in both tabs — verify-only):
```
grid grid-cols-1 sm:grid-cols-2 gap-4
```

Breakpoint: `sm` = 640px. Single-column below 640px, 2-column at and above 640px. Tailwind v4 `sm:` prefix unchanged.

**Vaccination list mode**: `grid grid-cols-1 gap-4` — always single-column regardless of viewport. Mobile behavior is already correct.

### Overscroll Lock (MOB-07)

**Class**: `overscroll-none` on the `<Card>` root element of `WallecxApp.vue`.

Generates: `overscroll-behavior: none` — prevents iOS rubber-band and pull-to-refresh.

Applies to Wallecx shell only. Does NOT affect other Lexarium mini-apps.

## Anti-Patterns

The executor must avoid these patterns specifically confirmed as wrong for this phase:

1. **`<style scoped>` for the dialog override** — scoped adds `[data-v-*]` which will not match PrimeVue's internal `.p-dialog-content` DOM node. Must be `<style>` with no scoped attribute.
2. **`overscroll-behavior: none` on `<body>` or `<html>` globally** — breaks pull-to-refresh on all Lexarium pages. Must be on `WallecxApp.vue` root only.
3. **`viewport-fit=cover` without safe-area padding** — causes content to be clipped behind iPhone notch. The three changes (viewport meta + App.vue top inset + WallecxApp.vue bottom/side insets) must ship together.
4. **iOS detection with only `/iPhone|iPad|iPod/` check** — Chrome on iOS also matches. Must exclude CriOS, FxiOS, OPiOS.
5. **`min-h-[44px]` on `<Card>` `#content` slot elements** — PrimeVue Card renders with its own slot wrapper. Apply the class to the `<Card>` component attribute, not to inner slot elements.
6. **Banner z-index above scan overlay** — `z-50` (50) is below scan overlay (`z-index: 9999`). Do not increase banner z-index above 50.

Source: RESEARCH.md Anti-Patterns section, Pitfalls 1–6.
