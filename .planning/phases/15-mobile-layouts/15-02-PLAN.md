  <action>
    Three changes in the template:

    **1. Search-clear InputIcon (lines 38-42) — wrap with a min-tap-target div:**

    The `InputIcon` element is not a `<button>` — it cannot receive `min-h`/`min-w` classes that produce a real tap target. Move the `@click` handler to a wrapper div.

    Replace:
    ```vue
    <InputIcon
      v-if="searchQuery"
      class="pi pi-times cursor-pointer"
      @click="emit('update:searchQuery', '')"
    />
    ```
    With:
    ```vue
    <div
      v-if="searchQuery"
      class="min-h-[44px] min-w-[44px] flex items-center justify-center touch-manipulation cursor-pointer"
      role="button"
      aria-label="Clear search"
      @click="emit('update:searchQuery', '')"
    >
      <InputIcon class="pi pi-times" />
    </div>
    ```

    **2. Sort Select (lines 44-51) — add min-h-[44px]:**

    The PrimeVue Select renders taller than 44px in most themes, but add the class to be explicit:
    ```vue
    <Select
      :model-value="sortMode"
      :options="sortOptions"
      option-label="label"
      option-value="value"
      class="w-36 min-h-[44px]"
      @update:model-value="emit('update:sortMode', $event)"
    />
    ```

    **3. View toggle Button (lines 52-66) — add min-h and min-w:**

    Replace the opening `<Button` tag's attributes to add the touch target classes (per D-12):
    ```vue
    <Button
      v-if="showToggle"
      severity="secondary"
      size="small"
      class="min-h-[44px] min-w-[44px] touch-manipulation"
      :aria-label="viewMode === 'grid' ? 'Switch to list view' : 'Switch to grid view'"
      :title="viewMode === 'grid' ? 'Switch to list view' : 'Switch to grid view'"
      @click="emit('update:viewMode', viewMode === 'grid' ? 'list' : 'grid')"
    >
    ```

    The iconify-icon child element is unchanged.
  </action>
