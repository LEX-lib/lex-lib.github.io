### 1. WallecxApp.vue (Shell — modified)

**Purpose:** Owns all state; wires list, dialog, and tokens.

**State refs added:**

async openDetail(record: Vaccinations): Promise<void>

**onMounted additions:**

### 4. AttachmentPreview.vue (New component)

**Props:**
```typescript
defineProps<{
  record: Vaccinations
  token: string
}>()
```

**MIME detection function:**
```typescript
function getMimeCategory(filename: string): 'image' | 'pdf' | 'unknown' {
  const lower = filename.toLowerCase()
  if (lower.endsWith('.jpg') || lower.endsWith('.jpeg') ||
      lower.endsWith('.png') || lower.endsWith('.webp')) return 'image'
  if (lower.endsWith('.pdf')) return 'pdf'
  return 'unknown'
}
```

**Computed values:**
```typescript
const mimeCategory = computed(() => getMimeCategory(props.record.card))

const tokenUrl = computed(() =>
  pb.files.getURL(props.record, props.record.card, { token: props.token })
)

const thumbUrl = computed(() =>
  pb.files.getURL(props.record, props.record.card, { thumb: '400x400', token: props.token })
)
```

**Async component:**
```typescript
const VuePdfEmbed = defineAsyncComponent(() => import('vue-pdf-embed'))
```

**Template — three branches:**

**Branch 1: Image**
```html
<img
  :src="thumbUrl"
  :alt="`${record.vaccine_name} vaccination card`"
  class="max-w-full rounded shadow-sm"
  style="max-height: 480px; object-fit: contain;"
/>
```
- Served via PocketBase `?thumb=400x400` — cropped/scaled to 400x400.
- `max-height: 480px` prevents oversized portrait images from pushing dialog off-screen.
- `object-fit: contain` preserves aspect ratio within the 400x400 bound.

**Branch 2: PDF**
```html
<Suspense>
  <VuePdfEmbed
    :source="tokenUrl"
    :page="1"
    @loading-failed="onPdfError"
    class="w-full"
  />
  <template #fallback>
    <div class="flex flex-col items-center py-6 gap-2">
      <Skeleton height="12rem" class="w-full" />
      <p class="text-sm" style="color: var(--color-typo-muted)">Loading PDF preview…</p>
    </div>
  </template>
</Suspense>
```
- Canvas width: `w-full` (fills dialog content width, ~560px at 40rem dialog).
- Only first page (`page="1"`) rendered in read-only view. Multi-page navigation is Phase 3+ scope.
- `@loading-failed` handler: sets local `pdfFailed = true` ref → shows download link fallback.

**PDF error fallback (pdfFailed = true):**
```html
<div class="flex flex-col gap-2">
  <p class="text-sm" style="color: var(--color-typo-muted)">
    Preview unavailable for this file.
  </p>
  <a :href="tokenUrl" download class="text-sm underline"
     style="color: var(--color-typo-link)">
    Download PDF
  </a>
</div>
```

**Branch 3: Unknown MIME**
```html
<div class="flex flex-col gap-2">
  <p class="text-sm" style="color: var(--color-typo-muted)">
    Preview not available for this file type.
  </p>
  <a :href="tokenUrl" download class="text-sm underline"
     style="color: var(--color-typo-link)">
    Download attachment
  </a>
</div>
```

**No attachment (record.card is falsy):**
```html
<p class="text-sm" style="color: var(--color-typo-muted)">No attachment.</p>
```

## Interaction & Motion

### List Row — Hover State

PrimeVue DataTable provides built-in row hover highlighting using the Aura preset's surface tokens. No custom hover CSS required. Row hover background: PrimeVue `surface-hover` token (light gray).

### "View" Button — Click

1. User clicks "View" button in DataTable row.
2. `emit('view', record)` fires immediately.
3. `WallecxApp.openDetail(record)` is called synchronously setting `selectedRecord`.
4. `pb.files.getToken()` is awaited (network call, ~200–400ms).
5. `showDetail.value = true` opens the Dialog after token resolves.
6. Dialog enters with PrimeVue's default overlay fade-in (CSS transition, ~200ms).
7. `VaccinationDetail` renders — AttachmentPreview MIME branch evaluates.
8. If PDF branch: `<Suspense>` fallback Skeleton renders while `vue-pdf-embed` async chunk loads.

**Token generation failure (network error during openDetail):**
- `toast.error("Failed to load attachment.")` fires.
- `showDetail.value = true` still executes — dialog opens showing all text fields.
- AttachmentPreview renders with empty `token` prop → `getURL` returns URL without token → browser 403 → image broken / PDF fails → fallback download link shown.
- This is acceptable degradation: user can still read all text fields.

### Dialog — Keyboard Navigation

- Escape key: closes Dialog (PrimeVue default behavior — `closeOnEscape: true` is default).
- Tab: cycles through focusable elements inside Dialog (close X button, download link if visible).
- Focus trap: PrimeVue Dialog traps focus inside the overlay when open.

### Dialog — Close

1. User clicks X button, clicks outside modal, or presses Escape.
2. `@hide` fires: `selectedRecord.value = null`, `fileToken.value = ''`.
3. Dialog fades out (PrimeVue default CSS transition ~200ms).
4. VaccinationList is visible and still mounted underneath (D-01 — no remount needed).

### Loading State (DataTable → Skeleton)

- `isLoading = true` immediately on `onMounted` entry.
- Skeleton DataTable renders: 3 rows, all 5 columns visible.
- Skeleton rows have no animation jitter — PrimeVue Skeleton uses a left-to-right shimmer by default.
- `isLoading = false` on `finally` → DataTable re-renders with real data or empty state. No flash: real data replaces skeleton in same DOM position.

### Attachment — PDF Loading

- Suspense fallback Skeleton (`height="12rem"`) renders immediately when PDF branch triggers.
- `vue-pdf-embed` async chunk loads (~350 kB, separate bundle split).
- Canvas renders page 1 at full dialog width.
- `@loading-failed`: transitions to download link fallback — no animation, immediate DOM swap.

### Toast Notifications

All toasts rendered by `<Toaster />` in App.vue. No Phase 2 changes to Toaster mount.

- Position: PrimeVue/vue-sonner default (top-right).
- Duration: vue-sonner default (4000ms auto-dismiss).
- `toast.error(...)` uses vue-sonner's error styling (red icon, dark background).
