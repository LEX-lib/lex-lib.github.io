const meetingData = {
  date: '2026-01-01',
  title: 'Standup',
  duration_minutes: 30,

const wrapper = mount(ManageMeeting, {

  },

To click the Save button, find it by label text since PrimeVue Button renders a `<button>` with the label inside:

// Option A: find by button text content
const saveBtn = wrapper.findAll('button').find(b => b.text().includes('Save Meeting'));

After click, assert the emit:

For ManageMeeting useDurationField round-trip test:
ManageMeeting seeds useDurationField with meeting.duration_minutes and meeting.duration_unit at component setup. It then watches [durationMinutes, unit] and writes back to meeting.value.duration_minutes and meeting.value.duration_unit. Because of the Vue reactivity watcher, after mount + flushPromises(), the meeting prop may already reflect the composable output.

Test case: mount with `{ duration_minutes: 90, duration_unit: 'hours' }`. The composable seeds enteredValue=1.5, unit='hours'. The output watcher writes 90/'hours' back to the model (unchanged). Click Save, verify emitted payload has duration_minutes=90 and duration_unit='hours'.

Import `flushPromises` from `@vue/test-utils` and await it after mount to let watchers settle before clicking Save.
