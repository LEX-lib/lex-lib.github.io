    ```typescript
    export interface Memberships extends RecordModel {
      id: string;
      created: string;
      updated: string;
      user: string;
      card_name: string;
      issuer?: string;

    ```

    Field type rationale (do NOT deviate):
    - `user: string` — Relation MaxSelect=1 returns the referenced record ID as a plain string
    - `card_name: string` — required field, no `?` modifier
    - `barcode_type?: string` — PocketBase SELECT fields serialize as a plain string (the selected value); typed as `string` not a union type to avoid compile errors if PocketBase returns an unexpected value; Phase 12 will use string equality checks
    - `expiry_date?: string` — PocketBase Date fields serialize as ISO strings; optional per schema
    - `card_color?: string` — hex WITHOUT leading `#`; CSS bindings in Phase 12 will prepend `#` via toCSS(hex) utility
    - `card_image?: string` — File MaxSelect=1 returns the filename as a plain string; `?` because PocketBase returns `""` when empty but the optional flag communicates intent cleanly; NOT `string[]`
    - `id`, `created`, `updated` are explicitly declared (inherited from RecordModel but explicit here matches the vaccinations analog)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| membershipMapper.ts return type → pb.update() | Inline return type enforces field stripping at compile time; user and card_image cannot appear in the PATCH payload |
| barcode_type typed as string → JsBarcode/qrcode.vue | Typing as string (not a union) means Phase 12 must validate barcode_type before passing to JsBarcode; invalid values throw synchronously — try/catch required per STATE.md risk register |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-14 | Tampering | membershipMapper including `user` field in update payload | mitigate | Inline return type explicitly excludes `user:`; TypeScript compiler rejects any code that tries to include it; prevents a client from re-assigning ownership of a card to another user |
| T-11-15 | Tampering | membershipMapper including `card_image` field in update payload | mitigate | `card_image` is absent from the inline return type; sending it in a standard update payload could overwrite the stored filename with an empty string, effectively orphaning the stored file |
| T-11-16 | Denial of Service | JsBarcode throws on invalid barcode_type input | accept | barcode_type typed as string; Phase 12 plan will wrap every JsBarcode() call in try/catch per STATE.md risk register (JsBarcode risk item BR-1); Phase 11 lays the type foundation only |
| T-11-17 | Information Disclosure | card_color stored with leading # | accept | card_color typed as plain string; convention (hex without #) is enforced by Phase 13 Zod validation `[0-9a-fA-F]{6}` and ColorPicker emit format; Phase 11 types are neutral on this |
