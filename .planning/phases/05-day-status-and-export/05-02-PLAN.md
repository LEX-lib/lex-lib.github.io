<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| PB record → banner display | `statusFullName` computed reads `dayStatus.value.status` from PB-returned record and maps to a hardcoded string — no PB string interpolated directly into HTML |
| user @change event → setDayStatus | `$event.value` from SelectButton is the `optionValue` from DAY_STATUS_OPTIONS — constrained to the 5 known string values or null |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-05 | Spoofing | Status banner content | accept | `statusFullName` computed uses a hardcoded lookup map — PB's raw status value is never interpolated directly into the DOM. Even if a PB value is unexpected, the fallback is `dayStatus.value.status` (raw string, not executable). Vue's template interpolation auto-escapes. |
| T-05-06 | Tampering | SelectButton `$event.value` | accept | SelectButton fires the `optionValue` from the bound `:options` array. The only values reachable via the UI are the 5 declared strings and null. PB enforces the enum server-side as the second line of defense (T-05-01). |
| T-05-07 | Information Disclosure | Banner shows selected date | accept | Date and status are user-chosen data the user explicitly set. No PII from third parties. Displayed only to the authenticated user on an auth-gated route. |
</threat_model>
