    Phase 5 adds two features to the LexTrack view at /projects/lextrack:
    1. Day Status SelectButton in the top row (SL / VL / Holiday / BL / Others options)
    2. Export Day button in the top row (copies formatted text to clipboard)
    When a status is set, the three activity columns are replaced by a centred status banner.
    Clearing the status restores the activity columns.
  </what-built>

  <how-to-verify>
Run `npm run dev` and navigate to http://localhost:5173/projects/lextrack (login if prompted).

**Smoke test 1 — Set day status:**
1. Observe the top row. You should see three groups: [DatePicker] [SelectButton: SL VL Holiday BL Others] [Export Day | Save].
2. Click "SL". Verify:
   - The three ActivityCard columns disappear.
   - A centred banner appears: "Sick Leave — YYYY-MM-DD" (today's date in YYYY-MM-DD format).
   - The SL button in SelectButton appears highlighted/active.

**Smoke test 2 — Clear day status:**
3. Click the active "SL" option again (deselect).
4. Verify:
   - The banner disappears.
   - The three ActivityCard columns reappear.
   - Any items you had in the lists before setting the status are still present (arrays preserved).

**Smoke test 3 — Status persists on date change:**
5. Set a status (e.g. VL) for today.
6. Change the date picker to a different date, then change back.
7. Verify the status banner for the original date reappears after switching back.

**Smoke test 4 — Export normal day:**
8. Clear any status (if set). Add at least one meeting, one task, and one admin item.
   - Meeting: title "Standup", duration 15
   - Task: title "Fix bug", jira_link "https://jira.example.com/TASK-1"
   - Admin: title "Code review"
9. Click "Export Day".
10. Verify a success toast appears: "Copied to clipboard!".
11. Paste into a text editor and confirm the format:
    ```
    [MM/DD/YYYY]
    Meetings:
    - Standup (15 mins)

    Tasks:
    https://jira.example.com/TASK-1 - Fix bug

    Admin:
    - Code review
    ```

**Smoke test 5 — Export status day:**
12. Set day status to "Holiday".
13. Click "Export Day".
14. Paste into text editor. Confirm the format is exactly two lines:
    ```
    [MM/DD/YYYY]
    Holiday
    ```
    (No section headers, no activity items.)
