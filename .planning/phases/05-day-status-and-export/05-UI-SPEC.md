## Interaction Contract

### Day Status — set

1. User clicks an inactive `SelectButton` option.
2. `setDayStatus(value)` fires immediately (D-03 — no separate Save click).
3. If a `dayStatus` record already exists for the date: call `pb.collection('dsu_day_status').update(existingId, payload)`.
4. If no record exists: call `pb.collection('dsu_day_status').create(payload)`.
5. On success: update `dayStatus` ref to the full PB record (stores `id` for future delete — D-18).
6. Template reacts: `v-if="!dayStatus"` hides activity grid; `v-else` shows banner.
7. Activity arrays (`meetings`, `tasks`, `supports`) are preserved in local state — NOT cleared.
8. No success toast for status set (action is immediate and visually self-confirming via banner).

### Day Status — clear

1. User clicks the currently active `SelectButton` option (`:allowEmpty="true"` allows deselection).
2. `setDayStatus(null)` fires.
3. Calls `pb.collection('dsu_day_status').delete(dayStatus.value.id)`.
4. On success: set `dayStatus` ref to `null`.
5. Template reacts: activity grid shown; banner hidden.
6. Pre-existing items re-appear (arrays were never cleared).
7. Error path: `toast.error("Couldn't clear day status — try again?")` and leave `dayStatus` unchanged (no optimistic rollback needed — UI didn't change yet).

Status clear is immediately reversible (re-select the same option) — no confirmation dialog.

### Day Status — load on date change

1. `loadForDate(date)` (extended from Phase 4) fires `Promise.all` with four requests including:
   ```
   pb.collection('dsu_day_status').getFullList({ filter: `date = "${YYYY-MM-DD}"` })
   ```
2. Result is a list of 0 or 1 records (unique constraint on date — D-08).
3. Assign: `dayStatus.value = statusList[0] ?? null`.
4. Uses the same `isLoading` flag as the other three entity fetches.

### Export — click

1. User clicks the Export Day button.
2. `exportDay()` handler builds the export string from current local state (no PB call).
3. Calls `navigator.clipboard.writeText(exportString)`.
4. On success: `toast.success('Copied to clipboard!')`.
5. On failure (clipboard API rejected): `toast.error('Copy failed — check browser permissions.')`.
6. Export Day button is NOT disabled during clipboard operation (instantaneous; no loading state needed).
7. Export works regardless of `dayStatus` value (D-19): if status is set, produces two-line format; otherwise produces full structured format.

### Loading state

- `isLoading` dims the activity grid (existing `opacity-50 pointer-events-none` class — unchanged).
- `SelectButton` and Export Day button are disabled while `isLoading` is true.
- Status banner (when visible) is NOT dimmed by `isLoading` — it is a read-only display.

## Export Format Contract

Canonical source: `dsu-format.md` at repo root. This section summarizes the rules for implementation — read `dsu-format.md` first.

### Normal day

```
[MM/DD/YYYY]
Meetings:
- {Title} ({duration_minutes} mins)

Tasks:
{jira_link} - {Title}
* {description bullet 1}
* {description bullet 2}

- {Title}
* {description bullet}

Admin:
- {link} - {Title}
- {Title}
* {description bullet}
```

Rules:
- Date header uses `dayjs(selectedDate.value).format('MM/DD/YYYY')` wrapped in brackets: `[MM/DD/YYYY]`.
- Section header (`Meetings:`, `Tasks:`, `Admin:`) only emitted if section has at least one item (D-16).
- A blank line separates each section from the next.
- Meeting line: `- {title} ({duration_minutes} mins)`. If `duration_minutes` is null/undefined, omit the parenthetical entirely: `- {title}`.
- Duration is always in minutes — no hours conversion (D-13). Example: `(60 mins)` not `(1 hr)`.
- Task line with jira_link: `{jira_link} - {title}` (no leading dash — matches `dsu-format.md` EXAMPLE).
- Task line without jira_link: `- {title}`.
- Task description: strip Quill HTML; each non-empty paragraph becomes `* {text}`. Skip empty descriptions.
- Admin line with link: `- {link} - {title}`.
- Admin line without link: `- {title}`.
- Admin description: same `*` bullet rule as Tasks.

### Status day

```
[MM/DD/YYYY]
{Full Status Name}
```

Two lines only. No section headers, no blank lines, no items. Full status name from the mapping table above (e.g. `Sick Leave`).

### HTML stripping (D-17 — Claude's discretion resolved)

Use `DOMParser` (browser-native, no new dependency):

```typescript
const stripHtml = (html: string): string[] => {
  const doc = new DOMParser().parseFromString(html, 'text/html');
  return Array.from(doc.body.querySelectorAll('p, li, div'))
    .map(el => el.textContent?.trim() ?? '')
    .filter(line => line.length > 0);
};
```

Returns an array of non-empty text lines, each of which becomes a `* {line}` bullet in the export. If `description` is null/undefined/empty string, produce no bullets.
