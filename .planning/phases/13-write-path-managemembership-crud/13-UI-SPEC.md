| Property | Value |

## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline color-swatch gap, label-to-input gap (`gap-1`) |
| sm | 8px | Compact field spacing |
| md | 16px | Default between form fields (`space-y-4`) |
| lg | 24px | Dialog body padding (PrimeVue default), section spacing |
| xl | 32px | Layout gaps in parent tab |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Touch targets (scan overlay close button, MembershipDetail footer buttons): minimum 44px height. Source: SCAN-03 implementation in `MembershipDetail.vue` (`w-12 h-12` = 48px).
- Edit-mode thumbnail: 100px × 100px fixed (`?thumb=100x100`). Not on the 4px scale — dictated by PocketBase thumb API constraint.

## Typography

All sizes in pixels. All typography uses `--font-family: "Rubik", sans-serif`.

| Role | Size | Weight | Line Height | CSS token |
|------|------|--------|-------------|-----------|
| Body | 14px (`text-sm`) | 400 | 1.5 | `--color-typo-body: #3d4a5c` |
| Label | 14px (`text-sm`) | 400 | 1.5 | `--color-typo-heading: #0d1117` |
| Dialog header | 16px (`text-base`, PrimeVue Dialog default) | 600 | 1.2 | PrimeVue Aura theme |
| Muted / helper | 14px (`text-sm`) | 400 | 1.5 | `--color-typo-muted: #6b7280` |

Source: `src/assets/base.css` — all four CSS custom properties confirmed. Matches pattern in `ManageVaccination.vue` and `MembershipDetail.vue`.

Rules:
- Field labels use `style="color: var(--color-typo-heading)"` — same as the existing analog.
- Field values in detail view use `style="color: var(--color-typo-body)"`.
- Helper text (selected filename, thumbnail hint) uses `style="color: var(--color-typo-muted)"`.
- Validation error messages rendered via PrimeVue `<Message severity="error" size="small" variant="simple">` — no custom typography override.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--color-surface-page: #f5f7fa` | Page background, Dialog backdrop |
| Secondary (30%) | `--color-surface-card: #ffffff` | Dialog body, form field backgrounds |
| Accent (10%) | `--color-brand-accent: #e89820` | Submit button (primary Button), link text |
| Destructive | `--color-status-error: #c0392b` | Delete button in MembershipDetail footer only |

Accent reserved for:
- The primary submit button inside ManageMembership.vue (`<Button type="submit">` — PrimeVue default primary maps to indigo; accent amber is used for the app-level branded CTA in the tab header "Add card" button).
- Expiry warning badge (`severity="warning"`) — uses amber via PrimeVue Aura warning semantic.

Destructive reserved for:
- Delete button in MembershipDetail.vue footer — `severity="danger"` (maps to `--color-status-error: #c0392b` via PrimeVue Aura).
- ConfirmDialog accept button — `acceptProps: { label: 'Delete', severity: 'danger' }`.
- No other elements use destructive color.

Card color binding rule: `card_color` is stored as a 6-character hex string without `#`. All display bindings prepend `#` in CSS: `:style="{ backgroundColor: '#' + record.card_color }"`. The ColorPicker default pre-fill is `002244` (navy). Source: D-01, D-03 in CONTEXT.md and `MembershipDetail.vue` lines 98–100 (confirmed pattern already implemented).

## Component Inventory

Components touched in this phase and their visual contracts:

### ManageMembership.vue (new)

**Dialog shell:**
- `<Dialog modal :style="{ width: '40vw' }" :breakpoints="{ '960px': '75vw', '641px': '92vw' }" :closable="!isSaving" @hide="onHide">`
- Header: "Add Card" (create mode) / "Edit Card" (edit mode). Source: D-12.
- No footer slot — submit button lives inside the form body (same pattern as ManageVaccination.vue).

**Form layout:**
- Direct `v-model` refs — NOT `@primevue/forms` or `zodResolver`. Source: D-07, locked by STATE.md due to PrimeVue issue #8135.
- `class="space-y-4"` — 16px between each field wrapper.
- Each field wrapped in `<div class="flex flex-col gap-1">` (4px label-to-input gap).

**Field order (locked by CONTEXT.md discretion section):**

| Order | Field | Component | Required | Notes |
|-------|-------|-----------|----------|-------|
| 1 | `card_name` | `<InputText fluid>` | Yes | Label: "Card Name *" |
| 2 | `barcode_type` | `<Select fluid>` | No | Label: "Barcode Type"; options list below |
| 3 | `barcode_value` | `<InputText fluid>` | No | Conditional: `v-if="barcode_type !== 'number'"` |
| 4 | `card_number` | `<InputText fluid>` | No | Label: "Card Number" |
| 5 | `card_color` | `<ColorPicker>` | No | Label: "Card Colour" (AU spelling); inline with hex swatch |
| 6 | `expiry_date` | `<DatePicker fluid dateFormat="dd M yy">` | No | Label: "Expiry Date" |
| 7 | `issuer` | `<InputText fluid>` | No | Label: "Issuer" |

| 9 | `card_image` | `<FileUpload mode="basic" :auto="false">` | No | Label: "Card Image" (create) / "Replace image" (edit) |
| — | Submit | `<Button type="submit" fluid>` | — | Label: "Add Card" (create) / "Save Changes" (edit) |

**barcode_type Select options (locked by CONTEXT.md specifics):**
```
[
  { label: 'QR Code',    value: 'qr'      },
  { label: 'Code 128',   value: 'code128' },
  { label: 'EAN-13',     value: 'ean13'   },
  { label: 'Code 39',    value: 'code39'  },
  { label: 'Number only',value: 'number'  }
]
```

**barcode_value conditional behavior (D-02):**
- Field disappears entirely (`v-if`, not `v-show`) when `barcode_type === 'number'`.
- On switch to 'Number only', the `barcode_value` ref is set to `''` immediately (watcher or `@update:modelValue` on the Select).
- No transition animation — instant hide/show.

**ColorPicker layout (field 5):**
- Label "Card Colour" (AU English spelling — matches `MembershipDetail.vue` line 95 "Card Colour").
- Layout: `<div class="flex items-center gap-3">` containing `<ColorPicker v-model="card_color" />` and a live preview swatch `<span class="inline-block w-8 h-8 rounded" :style="{ backgroundColor: '#' + card_color }">`.
- Swatch is 32px × 32px (w-8 h-8) — visually adjacent to picker.
- No clear/reset button. Color always has a value; defaults to `002244` for new records.
- ColorPicker emits 6-char hex without `#`. Store as-is. Never prepend `#` in the form ref.

**Edit-mode thumbnail (D-03):**
- Placement: directly above the FileUpload component (field 9), inside the same `flex flex-col gap-1` wrapper.
- Rendered as: `<img :src="thumbnailUrl" class="w-24 h-24 object-cover rounded" alt="Current card image">` — 96px × 96px rounded.
- `thumbnailUrl` = `pb.files.getURL(record, record.card_image, { thumb: '100x100', token })`.
- Below the thumbnail, helper text: `"Current image — select a new file to replace it"` — `text-sm` at `--color-typo-muted`.
- FileUpload label changes to "Replace image" (passed via `chooseLabel` prop).
- Thumbnail only shown when: `isEditMode && record.card_image`.

**isSaving guard (D-08):**
- All `<InputText>`, `<Select>`, `<DatePicker>`, `<Textarea>`, `<ColorPicker>`, `<FileUpload>` receive `:disabled="isSaving"`.
- Submit `<Button>` receives `:loading="isSaving" :disabled="isSaving"`.
- Dialog receives `:closable="!isSaving"`.

**Validation error display:**
- Each required field (`card_name`) has: `<Message v-if="cardNameError" severity="error" size="small" variant="simple">{{ cardNameError }}</Message>` below the input.
- Error ref (`cardNameError`) set on failed `schema.safeParse()` call.
- Optional fields: no error Message element needed unless Zod finds a format violation (e.g. invalid hex in `card_color`).
- Error message copy: "Card name is required." — sentence-case, ends with period.

**pendingFile reset (from CONTEXT.md discretion):**
- Explicit `pendingFile.value = null` in the success branch before `visible.value = false`.
- Also reset in `onHide()` handler (covers user-closed-without-saving case).
- Mirrors HIGH-02 fix from Phase 3 review.

**New: footer action buttons (D-04):**
- Location: PrimeVue Dialog `footer` slot in the parent (`MembershipsTab.vue`) that wraps the detail dialog. The detail content component itself emits `edit` and `delete` events — the footer buttons live in the Dialog's `#footer` slot at the tab level.
- Alternatively: if MembershipDetail.vue is rendered inside a Dialog in MembershipsTab, add a `<template #footer>` with the two buttons. Either approach is acceptable so long as the buttons are visually in the dialog footer row.

**Button layout (footer row):**
```
[ Edit ]  [ Delete ]
```
- "Edit" button: `<Button label="Edit" icon="pi pi-pencil" severity="secondary" @click="emit('edit')" />`
- "Delete" button: `<Button label="Delete" icon="pi pi-trash" severity="danger" @click="emit('delete')" />`
- Both are standard size (not `size="small"`).
- Justify: right-aligned (`justify-end` or PrimeVue Dialog default footer alignment).
- Gap between buttons: 8px (`gap-2`).

**Emits added:**
- `emit('edit')` — parent transitions to edit dialog.
- `emit('delete')` — parent triggers confirm flow.

## Interaction States

### Form fields
| State | Visual |
|-------|--------|
| Default | PrimeVue Aura default — border `#d1d5db`, white bg |
| Focus | PrimeVue Aura focus ring — indigo outline |
| Disabled (`isSaving=true`) | PrimeVue Aura disabled — reduced opacity, not interactive |
| Error | PrimeVue Aura invalid — red border (automatic on `invalid` prop if used, or via Message below) |

### Submit button
| State | Visual |
|-------|--------|
| Default | PrimeVue primary — indigo fill, white label |
| Loading (`isSaving=true`) | Spinner replaces icon; label unchanged; opacity reduced |
| Disabled | Same as loading — both `:loading` and `:disabled` set simultaneously |

### barcode_value field transition
- No animation. Uses `v-if` — DOM removal and insertion. Field slides into layout naturally via `space-y-4` reflow.

### ColorPicker
- Inline picker (not inline=false popup). Recommended: `<ColorPicker v-model="card_color" />` with default popup format.
- If PrimeVue ColorPicker defaults to popup mode, that is acceptable — swatch preview beside it shows live color regardless.

### Delete flow sequence
1. User taps "Delete" in MembershipDetail footer.
2. `emit('delete')` fires.
3. MembershipsTab calls `confirm.require(...)` — ConfirmDialog appears.
4. User confirms → `deleteCard(record)` runs: `isSaving` guard not needed here (delete does not use the form). Use a separate `isDeleting` ref if visual feedback on delete is needed; otherwise no guard (acceptable to match VaccinationsTab pattern where delete is not guarded separately).
5. On success: `records.value.splice(idx, 1)` + detail dialog closed + success toast.
6. On failure: no splice, error toast. Detail dialog stays open.

## Accessibility Notes

- All form labels must be associated with their inputs. Use `<label :for="fieldId">` + matching `:inputId` prop, or wrap label and input in the same `flex flex-col` container (implicit association). The analog ManageVaccination.vue uses the wrapper approach — replicate it.
- ColorPicker: add `aria-label="Card colour picker"` on the `<ColorPicker>` element.
- Color swatch preview: `aria-hidden="true"` — decorative.
- ConfirmDialog: PrimeVue handles `role="dialog"` and focus trap automatically.
- Delete button in detail footer: `aria-label="Delete card"` to disambiguate from any other Delete affordances.
- Edit button in detail footer: `aria-label="Edit card"`.
