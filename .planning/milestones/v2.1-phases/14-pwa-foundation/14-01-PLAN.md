        {

The SPA catch-all rewrite (`"/(.*)" → "/index.html"`) is required for Vue Router HTML5 history mode. This file must be committed in the same deployment as the first sw.js (STATE.md locked requirement: "vercel.json must ship in the same deployment as the first SW").

Do NOT add a `vercel.json` to `.gitignore` — it must be committed.

Create `public/wallecx-icon.svg`. This is a dedicated square 512×512 SVG for PWA icon generation — it is NOT a replacement for `branding_logo.svg`, which remains unchanged for the site header.

The icon source must be square so `@vite-pwa/assets-generator` produces correctly proportioned home screen icons (RESEARCH.md Pitfall 7: landscape source → letterboxed icons → unreadable on home screen).

Write the following SVG content:

```svg
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 512 512" width="512" height="512">
  <!-- Navy background -->
  <rect width="512" height="512" fill="#002244" rx="0"/>
  <!-- White W letterform — bold, centered, clean -->
  <text
    x="256"
    y="340"
    font-family="Arial, Helvetica, sans-serif"
    font-size="280"
    font-weight="700"
    text-anchor="middle"
    fill="#ffffff"
  >W</text>
</svg>
```

Design rationale:
- `#002244` (navy) matches `theme_color` and `background_color` in the manifest — the splash screen and icon share the same background, providing visual continuity on install
- White "W" on navy is legible at 192×192px and 512×512px with adequate contrast
- No rounded corners on the rect — the OS applies its own icon shape mask (circular on iOS, squircle on Android); `rx="0"` avoids double-masking artifacts
- The `minimal-2023` preset in `@vite-pwa/assets-generator` applies the maskable safe-zone padding to `maskable-icon-512x512.png` automatically — the SVG does not need to pre-pad for the safe zone

This file is consumed only by the CLI generator (Plan 02 Task 1). It is committed to `public/` so it is available in CI.
