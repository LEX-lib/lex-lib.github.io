- Vite build error → read the error output and fix the reported configuration issue

After a successful build, run these verification checks:

**Check 1 — sw.js exists and does NOT auto-activate:**
```
node -e "const fs=require('fs'); const sw=fs.readFileSync('./dist/sw.js','utf8').slice(0,5000); console.log({exists:fs.existsSync('./dist/sw.js'), hasSkipWaiting:sw.includes('skipWaiting'), hasClientsClaim:sw.includes('clientsClaim'), hasNavigateFallback:sw.includes('index.html')})"
```
Expected: `exists: true`, `hasNavigateFallback: true`.
Note: With `registerType: 'prompt'`, Workbox generates a controlled `skipWaiting` that is only triggered on message receipt (not automatically). `hasClientsClaim` may also appear — this is normal for Workbox; the key is that `registerType: 'prompt'` ensures the SW never calls `self.skipWaiting()` unconditionally.

**Check 2 — manifest values are correct:**
```
node -e "const m=require('./dist/manifest.webmanifest'); console.log({name:m.name,scope:m.scope,start_url:m.start_url,display:m.display,theme_color:m.theme_color,icons:m.icons.length})"
```
Expected output:
```
{ name: 'Wallecx', scope: '/', start_url: '/projects/wallecx', display: 'standalone', theme_color: '#002244', icons: 3 }
```
- `scope` MUST be `"/"` — NOT `"/projects/wallecx"` (STATE.md locked decision)
- `icons` must be 3 (any 192, any 512, maskable 512)

**Check 3 — manifest link auto-injected (not manually added):**
```
node -e "const fs=require('fs'); const h=fs.readFileSync('./dist/index.html','utf8'); console.log({hasManifestLink:h.includes('rel=\"manifest\"'), autoInjected:h.includes('manifest.webmanifest')})"
```
Expected: `hasManifestLink: true` and `autoInjected: true` — confirms vite-plugin-pwa injected it (not a manual `index.html` edit).

**Check 4 — sw.js not cached by dist output (Vercel handles this via vercel.json):**
Confirm `vercel.json` exists at project root:
```
node -e "console.log(require('fs').existsSync('./vercel.json'))"
```

**Check 5 — no `<link rel="manifest">` in source index.html (only in dist):**
```
node -e "const fs=require('fs'); const src=fs.readFileSync('./index.html','utf8'); console.log('source index.html manifest link:',src.includes('rel=\"manifest\"'))"
```
Expected: `false` — vite-plugin-pwa injects only into `dist/index.html`, not the source.
