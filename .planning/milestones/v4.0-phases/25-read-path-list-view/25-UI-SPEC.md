## Spacing Scale

Declared values (multiples of 4 only). Source: Tailwind CSS 4 defaults applied throughout existing Wallecx components.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps within a row cell (`gap-1`) |
| sm | 8px | Gap between toolbar controls (`gap-2`); row inner padding adjustments |
| md | 16px | Default horizontal padding for row content (`px-4`); gap between major row sections |
| lg | 24px | Vertical empty-state padding (`py-6`); toolbar bottom margin (`mb-4`) |
| xl | 32px | N/A in this phase |
| 2xl | 48px | Empty-state vertical padding (`py-12`) ‚Äî matches MembershipsTab pattern |
| 3xl | 64px | N/A in this phase |

Exceptions:
- **Touch targets:** All icon buttons (paperclip, pencil, trash) must be `min-w-[44px] min-h-[44px]` ‚Äî Phase 15 invariant. This is a 44px floor, not a spacing token.
- **Row vertical rhythm:** `py-3` (12px top + bottom) per `ExpenseItem` row ‚Äî a deliberate non-8px value chosen in CONTEXT.md to keep rows compact. Accepted exception.
- **Amount column width:** fixed `w-24` (96px) ‚Äî layout constraint, not a spacing token.

## Interaction Contracts

### Loading state
- While `isLoading` is `true`: render 3 skeleton rows using `<Skeleton height="3rem" class="w-full rounded" />` inside a `<div class="flex flex-col gap-1">` wrapper.
- Row height target: 3rem (48px) per skeleton ‚Äî matches the approximate rendered height of a compact `ExpenseItem` row with two lines of text.
- Source: CONTEXT.md D-17; mirrors MembershipsTab skeleton pattern.

### Empty state ‚Äî no records
Condition: `expenses.length === 0 && !isLoading`

| Element | Spec |
|---------|------|
| Icon | `mdi:cash-multiple` at 48√ó48px, `color: var(--color-brand-primary)` |
| Heading | `"No expenses yet."` ‚Äî `text-sm font-bold`, `color: var(--color-typo-heading)` |
| Body | `"Add your first expense to start tracking."` ‚Äî `text-sm`, `color: var(--color-typo-muted)` |
| CTA button | `"Add expense"` ‚Äî PrimeVue `Button` size="small", primary severity |
| Layout | `flex flex-col items-center py-12 gap-3` |

Source: CONTEXT.md D-12 + Phase 24 stub template (already uses `mdi:cash-multiple`).

### Empty state ‚Äî filters produce no results
Condition: `expenses.length > 0 && filteredSortedExpenses.length === 0`

| Element | Spec |
|---------|------|
| Icon | `mdi:filter-remove-outline` at 48√ó48px, `color: var(--color-brand-primary)` |
| Heading | `"No expenses match your filters."` ‚Äî `text-sm font-bold`, `color: var(--color-typo-heading)` |
| CTA button | `"Clear filters"` ‚Äî PrimeVue `Button` severity="secondary" size="small" |
| Layout | `flex flex-col items-center py-12 gap-3` |

Source: CONTEXT.md D-12.

### Error state ‚Äî load failure
Triggered when `getFullList` throws in `onMounted`.

- `vue-sonner` toast: `toast.error('Failed to load expenses. Pull to refresh or reload the page.')` (mirrors MembershipsTab pattern)
- No inline error UI ‚Äî toast is sufficient; list remains empty (not loading).

### Receipt preview flow
1. User taps paperclip icon on a row with `record.receipt` set.
2. `openReceiptPreview(record)` calls `pb.files.getToken()`.
3. On token success: set `previewRecord.value = record`, `previewToken.value = token`, `showPreview.value = true`.
4. On token failure: `toast.error('Failed to load receipt. Refresh to try again.')` ‚Äî do NOT open preview (mirrors MembershipsTab WR-03 pattern).
5. Preview rendered in a `<Dialog>` (desktop) or `<Drawer position="bottom">` (mobile, via `useIsMobile()`). `AttachmentPreview` component inside, props: `record=previewRecord`, `attachmentField="receipt"`, `attachmentName="Receipt"`, `token=previewToken`.

Source: CONTEXT.md D-13, D-14, D-15.

### Sort persistence
- sessionStorage key: `wallecx:expense-sort` (locked in EXP-07)
- Default on first visit: `newest-first`
- Valid modes whitelist: `['newest-first', 'oldest-first', 'category-asc', 'amount-high', 'amount-low']`
- Restore on `onMounted` before `getFullList` (mirrors MembershipsTab lines 82‚Äì92 pattern)
- Persist on `watch(sortMode)` (mirrors MembershipsTab lines 74‚Äì80 pattern)

Source: CONTEXT.md D-03, D-04.

### Filter/sort order
Applied in this sequence inside `filteredSortedExpenses` computed (source: CONTEXT.md D-10):
1. Description search (case-insensitive `includes`)
2. Category filter (`selectedCategories` ‚Äî empty array = show all)
3. Date range (`dayjs` comparison, both bounds inclusive)
4. Sort

### Debounce
Search input debounced at **300ms** using `useDebounce` from `@vueuse/core`. Source: CONTEXT.md D-09.

### Touch targets
Every interactive element (icon buttons, toolbar controls, Select, MultiSelect, DatePicker) must have `min-h-[44px]`. Source: Phase 15 invariant, CONTEXT.md D-02.
