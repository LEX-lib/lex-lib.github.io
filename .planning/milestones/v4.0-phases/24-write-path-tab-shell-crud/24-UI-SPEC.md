## Spacing Scale

Declared values (all multiples of 4). Inherited from Phase 13 UI-SPEC — no changes for this phase.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, label-to-input gap (`gap-1`) |
| sm | 8px | Compact field spacing, footer button gap (`gap-2`) |
| md | 16px | Between form fields (`space-y-4`), tab header bottom margin (`mb-4`) |
| lg | 24px | Dialog body padding (PrimeVue Aura default), section spacing |
| xl | 32px | Layout gaps in parent card shell |
| 2xl | 48px | Empty state vertical padding (`py-12`) |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Touch targets (dialog action buttons on mobile, add button): minimum 44px height. Applies via `size="small"` PrimeVue Button which meets the 44px threshold (confirmed in Phase 15 implementation).
- Edit-mode receipt thumbnail: 96px × 96px (`w-24 h-24`) — dictated by PocketBase thumb API; not on 4px scale.

## Typography

All sizes in pixels. All typography uses `font-family: "Rubik", sans-serif`.

| Role | Size | Weight | Line Height | CSS token |
|------|------|--------|-------------|-----------|
| Body | 14px (`text-sm`) | 400 | 1.5 | `--color-typo-body: #3d4a5c` |
| Label | 14px (`text-sm`) | 400 | 1.5 | `--color-typo-heading: #0d1117` |
| Dialog header | 16px (`text-base`, PrimeVue Dialog default) | 600 | 1.2 | PrimeVue Aura theme |
| Muted / helper | 14px (`text-sm`) | 400 | 1.5 | `--color-typo-muted: #6b7280` |

Source: `src/assets/base.css` — all four CSS custom properties confirmed present. Matches the pattern established in `ManageVaccination.vue`, `ManageMembership.vue`, and Phase 13 UI-SPEC.

Rules:
- Field labels: `class="text-sm" style="color: var(--color-typo-heading)"` — exact match to ManageMembership.vue pattern.
- Helper text (filename hint, thumbnail hint, char counter): `class="text-sm" style="color: var(--color-typo-muted)"`.
- Validation error messages: `<Message severity="error" size="small" variant="simple">` — PrimeVue built-in; no custom typography override.
- Description character counter (120 char max): `class="text-xs" style="color: var(--color-typo-muted)"` — right-aligned below the InputText.

Dark mode: All `var(--color-typo-*)` tokens remap automatically under `.my-app-dark` via `base.css` overrides (confirmed in Phase 22 audit).

"PHP" label is a static `<span>` to the left of the InputNumber — not inside the field, not using PrimeVue currency mode (avoids locale/keyboard quirks on mobile per D-13).

**Description field with character counter:**
```html
<div class="flex flex-col gap-1">
  <label class="text-sm" style="color: var(--color-typo-heading)">Description *</label>
  <InputText v-model="description" fluid :maxlength="120" :disabled="isSaving" />
```

Counter shows `{current}/120` in muted color, right-aligned. Updates live as the user types.

**Category field loading state:**
- While `isLoadingCategories` is true: Select is disabled with placeholder `"Loading categories…"` (D-22).
- `<Select :placeholder="isLoadingCategories ? 'Loading categories…' : 'Select or type a category'">`

**Receipt field — edit-mode thumbnail:**
- Placement: directly above the FileUpload component, inside the same `flex flex-col gap-1` wrapper.
- Rendered as: `<img :src="receiptThumbnailUrl" class="w-24 h-24 object-cover rounded" alt="Current receipt">` — 96×96px rounded.
- `receiptThumbnailUrl` = `pb.files.getURL(record, record.receipt, { thumb: '100x100', token })` — only for image files (JPEG/PNG/WebP). PDF shows a paper-icon placeholder instead (see copy table).
- Helper text: `"Current receipt — select a new file to replace it"` — `text-sm` at `--color-typo-muted`.
- FileUpload `chooseLabel` changes to `"Replace file"` in edit mode when `record.receipt` exists.
- PDF files skip EXIF strip (D-17) — only JPEG/PNG/WebP go through the canvas re-encode + `browser-image-compression` pipeline.

**File type validation** (in `onFileSelect` handler):
- Accepted: `image/jpeg`, `image/png`, `image/webp`, `application/pdf`.
- Max size: 10 MB (`10 * 1024 * 1024` bytes).
- Images: EXIF strip via canvas re-encode → `browser-image-compression` (maxSizeMB: 1.5, maxWidthOrHeight: 2048, useWebWorker: true).
- PDF: no EXIF processing — assign directly to `pendingFile.value`.

**isSaving guard** (D-08 — all fields + submit):
- All `<InputNumber>`, `<DatePicker>`, `<Select>`, `<InputText>`, `<Textarea>`, `<FileUpload>` receive `:disabled="isSaving"`.
- Submit `<Button>` receives `:loading="isSaving || isLoadingCategories" :disabled="isSaving || isLoadingCategories"`.
- Dialog / Drawer receives `:closable="!isSaving && !isLoadingCategories"`.

**Validation error display** (D-09 — Zod `expenseSchema.safeParse()`):
- Call `expenseSchema.safeParse(payload)` on submit.
- Map `result.error.flatten().fieldErrors` to per-field error refs.
- Error message copy — sentence-case, ends with period (mirrors ManageMembership pattern):
  - `amountError` → "Amount must be greater than 0."
  - `expenseDateError` → "Date is required."
  - `categoryError` → "Category is required."
  - `descriptionError` — "Description is required." (empty) or "Description must be 120 characters or fewer." (overflow).

**Amount rounding on save:** `Math.round(amount.value! * 100) / 100` before appending to FormData. Matches D-13.

**Date serialization on save:** `dayjs(expenseDate.value).format('YYYY-MM-DD')`. Matches D-14.

**Category new-category logic on submit** (D-18, D-19):
1. Check if `category.value` matches any name in `loadedCategories.value`.
2. If not found: `await pb.collection('wallecx_expense_categories').create({ user: userId, name: category.value })`.
3. If category creation fails: abort expense save, show toast error, keep dialog open with data intact (D-20).
4. Then proceed with expense create/update.

**Auth null guard** (D-11):
```ts
const userId = pb.authStore.record?.id;
if (!userId) {
  toast.error('Session expired. Please log in again.');
  isSaving.value = false;
  return;
}
```

**Category seed on dialog open** (D-21, D-22):
1. Load `wallecx_expense_categories` with `requestKey: 'expense-categories-getFullList'`.
2. If result is empty (length === 0): set `isLoadingCategories.value = true`, seed 7 defaults via `Promise.all`, reload, set `isLoadingCategories.value = false`.
3. Submit button is disabled while `isLoadingCategories` is true.

**requestKey values** (D-12, STATE.md locked):
- Expense list load: `{ requestKey: 'expenses-getFullList' }`
- Category load: `{ requestKey: 'expense-categories-getFullList' }`

**Emits:**
- `created: [record: Expenses]`
- `updated: [record: Expenses]`

**`onHide` handler:**
- Resets `pendingFile.value = null` (HIGH-02 pattern — also reset in success branch before `visible.value = false`).

### Delete flow sequence (Phase 24 — triggered from ExpensesTab)

1. User taps "Delete" action (available in Phase 25 edit/detail view; Phase 24 wires the confirm flow ready for Phase 25 to call).
2. `confirm.require(...)` fires → ConfirmDialog at WallecxApp.vue shell level appears.
3. Header: "Confirm Delete" / Message: "Delete this expense? This cannot be undone." / Icon: `pi pi-exclamation-triangle`.
4. User confirms → `deleteExpense(record)` runs: `pb.collection('wallecx_expenses').delete(record.id)` (server-first, D-10).
5. On success: `expenses.value.splice(idx, 1)` + success toast.
6. On failure: no splice, error toast. Dialog stays closed (confirm already dismissed).

### Create/edit flow sequence

1. User taps "Add Expense" (header button or empty state button).
2. Dialog/Drawer opens. If `isLoadingCategories` is true (first open ever), category Select shows disabled with loading placeholder; Submit disabled.
3. Seeding completes → Select enabled with populated options.
4. User fills form. Description counter updates live.
5. User taps Submit → `expenseSchema.safeParse()` runs:
   - Failures: per-field error Messages shown, form stays open.
   - Success: `isSaving.value = true`, all fields disabled, closable=false.
6. If category is new: category created first (D-19). On category create failure: abort, toast error, `isSaving.value = false`, dialog stays open.
7. Expense create/update call executes. On success: emit `created`/`updated`, `pendingFile` reset, dialog closed, success toast. On failure: error toast, `isSaving.value = false`, dialog stays open.

## Accessibility Notes

- All form labels must be associated with their inputs via wrapper approach (`<div class="flex flex-col gap-1">` containing label then input — matches ManageMembership.vue pattern).
- Amount InputNumber: `aria-label` not needed — label element in the same wrapper provides association.
- PHP currency span: `aria-hidden="true"` — decorative; the label "Amount *" provides the field identity.
- Description character counter: `aria-live="polite"` on the counter span so screen readers announce updates without interrupting typing.
- Category Select with `editable=true`: PrimeVue's editable Select renders as an input — ensure the wrapping label provides the accessible name.
- FileUpload: PrimeVue `mode="basic"` renders a button; `chooseLabel` prop sets the button text and thus the accessible name.
- Receipt thumbnail (edit mode): `alt="Current receipt"` (image) or `alt="Receipt (PDF)"` (PDF placeholder icon).
- Drag handle pill (mobile Drawer): `aria-hidden="true"` — decorative.
- ConfirmDialog: PrimeVue handles `role="alertdialog"` and focus trap automatically.
- Tab icon (`mdi:cash-multiple`): `aria-hidden="true"` — tab label text provides the accessible name.
- isSaving disabled state: `aria-disabled="true"` is set automatically by PrimeVue when `:disabled="true"` is bound.
