# Milestone v2.0: Membership Cards

**Status:** ✅ SHIPPED 2026-05-14
**Phases:** 10–13
**Total Plans:** 12

## Overview

Wallecx expanded from a single-record-type vault into a dual-tab vault. The vaccination tracker became `VaccinationsTab.vue` inside a PrimeVue Tabs shell, and a full membership/loyalty/insurance/ID card system was built alongside it — with coloured card tiles, barcode and QR rendering, a full-screen scan overlay for counter use, and complete CRUD (create, edit, delete) protected by the same per-user PocketBase isolation pattern as vaccinations.

## Milestone Summary

### Key Accomplishments

1. **Tabs shell extraction** — WallecxApp.vue refactored from a 452-line vaccination monolith into a 35-line PrimeVue Tabs shell; all vaccination state, computed refs, lifecycle hooks, and template verbatim-extracted into a self-contained VaccinationsTab.vue with zero regression
2. **Per-user membership backend** — `wallecx_memberships` PocketBase collection with 10 fields, 5 per-user access rules, image-only protected card_image file field; two-user smoke test confirmed cross-user isolation across list/view/update/delete/file access types
3. **Barcode & QR rendering** — BarcodeDisplay.vue four-branch computed (qr/linear/number-fallback/empty) with JsBarcode try/catch guard; qrcode.vue SVG render; BARCODE_FOREGROUND/BACKGROUND constants; renderError state prevents invalid-input crashes
4. **Full-screen scan overlay** — Teleport + `position:fixed;inset:0;z-index:9999` viewport overlay (iOS-safe, no Fullscreen API); `filter:brightness(1.4)`; `navigator.wakeLock.request('screen')` with feature-detect + visibilitychange re-acquire; always-visible close button
5. **ManageMembership.vue CRUD dialog** — Direct v-model refs (PrimeVue ColorPicker issue #8135 workaround); Zod safeParse; conditional barcode_value field; EXIF-stripped FileUpload; `isSaving` guard; server-first delete; Object.assign id-refresh on create to prevent save-loop bug
6. **Test coverage** — membershipMapper.spec.ts Vitest spec locking 5-field strip and id-refresh contract (11 tests); 24 tests total passing across the repo

### Key Decisions

- **PrimeVue Tabs, not sub-routes** — Sub-routes would break existing `/projects/wallecx` bookmarks and require a non-trivial router restructure. String-typed `activeTab` ref bound to `v-model:value` on `<Tabs>`.
- **Each tab owns its own state** — VaccinationsTab.vue and MembershipsTab.vue each own independent local refs. WallecxApp.vue holds only `activeTab`. No new Pinia store.
- **ManageMembership.vue uses direct v-model refs** — PrimeVue issue #8135 (unfixed): ColorPicker inside a PrimeVue Form always starts showing red. Direct v-model pattern avoids the controlled-system bug.
- **card_color stored without `#` prefix** — Matches ColorPicker emit format. All CSS bindings prepend `#` via `toCSS(hex)` utility. Zod validates `[0-9a-fA-F]{6}`.
- **iOS fullscreen via viewport overlay** — `requestFullscreen()` unsupported on non-video elements on iOS < 26. Fixed inset overlay works on all devices.
- **Every JsBarcode() call wrapped in try/catch** — JsBarcode has no soft-fail mode; invalid input throws synchronously. On catch, renders `card_number` as large plain text.
- **ConfirmDialog kept at WallecxApp.vue shell level** — `useConfirm` broadcasts to the single app-shell-level ConfirmDialog instance. No duplicate tags in tab components.
- **requestKey: 'memberships-getFullList'** — Distinct from vaccinations key to prevent PocketBase auto-cancel silently dropping parallel fetches.

### Issues Resolved

- PrimeVue ColorPicker inside controlled Form always shows red initial state (issue #8135) — worked around with direct v-model refs
- iOS fullscreen API does nothing on non-video elements — worked around with viewport overlay
- JsBarcode throws synchronously on invalid input — wrapped in try/catch with card_number plain-text fallback
- PocketBase auto-cancel silently drops parallel fetches — mitigated with distinct requestKey per collection
- Wake lock drops on tab visibility change — re-acquired on `visibilitychange` event
