  <what-to-do>
Open the PocketBase admin UI and add vaccine_type as a new field on the wallecx_vaccinations collection. Follow these exact steps:

1. Navigate to your PocketBase admin UI (typically http://127.0.0.1:8090/_/ for local dev, or your hosted instance URL).

2. In the left sidebar, click **Collections**.

3. Click **wallecx_vaccinations** to open the collection editor.

4. Click the **+ New field** button (or the equivalent field addition control).

5. Configure the new field as follows:
   - **Field type:** Text
   - **Field name:** `vaccine_type`  (exactly this string — no spaces, no capitals)
   - **Required:** UNCHECKED / OFF — the field must be optional at schema level (D-05, locked)
   - **Min / Max length:** leave blank (no constraints)
   - **Pattern:** leave blank
   - All other settings: leave at defaults

6. Click **Save** (or **Save field**, then **Save collection** — depending on your PocketBase version, there may be two save steps).

7. Verify the field was added:
   - The collection schema list shows `vaccine_type` as a Text field
   - The Required column shows it as NOT required

8. Verify existing records are not corrupted:
   - Open any existing vaccination record via the Records tab (or via the admin record browser)
   - Confirm the record loads without error
   - Confirm vaccine_type shows as empty string (not null, not an error)

**Why this must happen before going live:** 05-01-PLAN.md adds `formData.append("vaccine_type", ...)` to every create and update call. If the field does not exist in the collection, PocketBase rejects the request with a 400 error. The code changes can be deployed to the frontend, but no create/update will succeed until this step is done.

**Why optional at schema level (D-05):** Existing records were created before this field existed. Marking the field as required in PocketBase would cause those records to fail validation on any subsequent update. Zod in the form enforces required for users; PocketBase accepts empty string on old records.
