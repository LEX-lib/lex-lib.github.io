# Milestone v4.0: Daily Expense Tracker

**Status:** ✅ SHIPPED 2026-05-22
**Phases:** 23–26
**Total Plans:** 9

## Overview

Adds a third Wallecx record type — expenses — with daily logging, period-tabbed reporting (month / quarter / year / custom), and per-category breakdown charts. Expands Wallecx from static personal records (vaccinations, memberships) into time-series spending data.

## Milestone Summary

**Key Decisions:**
- category stored as denormalized Text (not Relation) to prevent retroactive history rewrite on category rename
- DEFAULT_EXPENSE_CATEGORIES defined in Phase 23, seeded lazily in Phase 24 on first dialog open (no PocketBase signup hook)
- Period selector uses PrimeVue Tabs (scrollable), not SelectButton — established PrimeVue pattern with natural overflow handling
- Custom From/To persisted as YYYY-MM-DD strings (not Date/ISO) for timezone-stable sessionStorage
- Chart palette fully reactive via useChartTheme refs — no manual chart remount on dark-mode toggle
- dayjs Q-format token NOT extended by quarterOfYear plugin — template literal Q${now.quarter()} used instead
- prefers-reduced-motion → animation.duration: 0 (200ms otherwise) — read inline in computed, no @vueuse/core dep
- Both sibling views (List + Reports) consume the same expenses + isLoading props from the shell; single getFullList preserved

**Issues Resolved:**
- PocketBase createRule uses @request.data.user (not @request.body.user) — confirmed and locked in memory
- expenseMapper WR-01 (conditional notes key) + WR-02 (strong assertion) fixed in Phase 24
- expenseSchema WR-03 (dayjs strict parse for calendar date validation) fixed in Phase 24
- dayjs Q-format empirical discovery corrected before shipping (template literal workaround)
- MutationObserver targets document.documentElement (not body) — project's useTheme.ts puts .my-app-dark on <html>

**Issues Deferred:**
- EXP-ADV-01..07: budget tracking, recurring expenses, multi-currency, period comparison, tags, OCR, JSON export
- Sub-tab persistence (activeSubTab sessionStorage) — deferred per UI-SPEC; List is always default on tab entry
- Phase 25 UAT partial scenarios (7 pending) — deferred to maintenance sweep

**Technical Debt Incurred:**

- LexTrack DatePicker/TabView contrast and API Playground sign-off from Phase 21 — still deferred
