## Spacing Scale

Declared values (multiples of 4 only — inherits from Phase 2 baseline):

| Token | Value | Usage in Phase 12 |
|-------|-------|-------------------|
| xs | 4px | Icon-to-text gap (`gap-1`), badge internal padding, barcode-type chip gap |
| sm | 8px | Between card tile internal elements (`gap-2`), overlay close button margin |
| md | 16px | Card tile padding, dialog body padding, field label/value gap, BarcodeDisplay internal padding |
| lg | 24px | Dialog section spacing, empty-state vertical padding (`py-6`), BarcodeDisplay white container padding |
| xl | 32px | Card grid gap on wider screens, MembershipDetail section spacing |
| 2xl | 48px | Empty-state vertical center padding (`py-12`), overlay barcode centering |
| 3xl | 64px | Not used in Phase 12 |

Exceptions:
- Card tile minimum height: `min-h-[8rem]` (128px) — ensures tiles with only card_name still feel substantial.
- Scan overlay barcode container: `max-w-xs` (320px) — constrains barcode width for scanner readability.
- Overlay close button touch target: `w-12 h-12` (48px square) — accessibility minimum for fingertip use.
- BarcodeDisplay white container: `p-6` (24px all sides) — BARCODE_BACKGROUND #ffffff container must have adequate margin around the barcode image.

## Typography

Font family: Rubik, sans-serif (loaded via `main.css`, available throughout).

| Role | Size | Weight | Line Height | Token / Class | Used For |
|------|------|--------|-------------|---------------|----------|
| Body | 14px | 400 (Regular) | 1.5 | `text-sm` | Card tile issuer text, expiry date, field values in detail, field labels in detail, toast messages, badge text |
| Card heading | 18px | 700 (Bold) | 1.2 | `text-lg font-bold` | Card tile card_name (primary text per tile) |
| Barcode fallback | 28px | 700 (Bold) | 1.2 | `text-3xl font-bold` | card_number rendered as large plain text when barcode unavailable |
| Page heading | 24px | 700 (Bold) | 1.2 | `text-2xl font-bold` | "Wallecx" `<h1>` in WallecxApp.vue — existing, unchanged |

Notes:
- Field labels in MembershipDetail use `var(--color-typo-heading)` (#0d1117); field values use `var(--color-typo-body)` (#3d4a5c). Both are `text-sm` / weight 400. Color contrast is the sole visual distinguisher — matches Phase 2 VaccinationDetail pattern exactly.
- The barcode fallback text (card_number at 28px Bold) is intentionally oversized — it replaces a barcode image and must be legible from counter-scanning distance.
- Semibold (600) is not used. Rubik's standard weight set is 300, 400, 500, 700, 900. Matches Phase 2 decision.
- Card tile issuer text: `text-sm` weight 400, `var(--color-typo-muted)` — visually subordinate to card_name.
- "Barcode unavailable" caption beneath fallback card_number: `text-sm` weight 400, `var(--color-typo-muted)`.
- "No barcode" placeholder: `text-sm` weight 400, `var(--color-typo-muted)` — centered in BarcodeDisplay container.
- Scan overlay: barcode_value / card_number rendered at the same 28px Bold size as BarcodeDisplay fallback text — overlay merely enlarges it full-screen at maximum scan size.

| Role | Value | Token | Usage in Phase 12 |
|------|-------|-------|-------------------|
| Dominant (60%) | `#f5f7fa` | `--color-surface-page` | Page background behind the Card wrapper |
| Secondary (30%) | `#ffffff` | `--color-surface-card` | WallecxApp `<Card>` surface; Dialog modal surface; BarcodeDisplay white container (BARCODE_BACKGROUND) |

| Accent warm | `#e89820` | `--color-brand-accent` / `--color-status-warning` | Expiry warning badge background (#e89820 / warning severity); no other usages in Phase 12 |
| Text heading | `#0d1117` | `--color-typo-heading` | Card tile card_name (when tile background is white), field labels in detail |
| Text body | `#3d4a5c` | `--color-typo-body` | Field values in MembershipDetail |
| Text muted | `#6b7280` | `--color-typo-muted` | Card tile issuer, expiry text, "No attachment.", "Barcode unavailable", "No barcode" placeholder |
| Text on colored tiles | `#ffffff` | (inline) | card_name + issuer on coloured card_color tile backgrounds (always white for contrast) |
| Barcode foreground | `#000000` | `BARCODE_FOREGROUND` constant | JsBarcode lineColor + QR foreground. Never configurable. |
| Barcode background | `#ffffff` | `BARCODE_BACKGROUND` constant | BarcodeDisplay white container. Wraps both SVG branches. Non-negotiable. |
| Error | `#c0392b` | `--color-status-error` | Toast error border/icon tint (vue-sonner handles styling) |
| Scan overlay | `#ffffff` | (inline) | `position: fixed; inset: 0; background: #ffffff` with `filter: brightness(1.4)` |

**card_color tile text:** All text on coloured tiles (card_name, issuer, barcode_type badge, expiry date, expiry warning badge) uses `color: #ffffff` regardless of card_color value. There is no per-card contrast calculation — white-on-colour works for the dark-ish default navy and covers the range of reasonable loyalty card colours. Light pastel cards are an edge case acceptable for v2.0.

**card_color CSS binding rule (locked — STATE.md):**
```
:style="{ backgroundColor: card.card_color ? '#' + card.card_color : '#002244' }"
```
The `toCSS(hex)` utility in the codebase prepends `#`. Every CSS background binding uses this pattern. Never pass raw card_color to CSS.

**Accent reserved for:**
- PrimeVue primary `<Button>` fills (navy #002244 in light mode, amber #e89820 in dark mode via MyPreset)
- Empty-state `iconify-icon` color (`style="color: var(--color-brand-primary)"`)
- Default card tile background when `card_color` is absent
- Tab icon in WallecxApp.vue Tab component

## Component Specifications

### 1. BarcodeDisplay.vue (New — purely presentational)

**Purpose:** Renders a barcode or QR code on a white background for embedding in MembershipCard (small) and MembershipDetail (full-size). Emits a `scan` event when tapped. No PocketBase calls.

**Props:**
```typescript
defineProps<{
  barcode_type: string | undefined   // 'qr' | 'code128' | 'ean13' | 'code39' | 'number' | undefined
  barcode_value: string | undefined  // raw barcode string; may be empty or undefined
  card_number: string | undefined    // plain fallback number; may be empty or undefined
}>()
```

**Emits:**
```typescript
defineEmits<{
  scan: []  // fired when user taps/clicks the barcode area to open the scan overlay
}>()
```

**Constants (module-level, not props — locked by STATE.md):**
```typescript
const BARCODE_FOREGROUND = '#000000'
const BARCODE_BACKGROUND = '#ffffff'
```

**Render logic — four mutually exclusive branches (evaluated top to bottom):**

**Branch A — QR code (barcode_type === 'qr' AND barcode_value truthy):**
```
<div class="bg-white p-6 rounded flex items-center justify-center cursor-pointer"
     @click="emit('scan')">
  <QrcodeVue
    :value="barcode_value"
    :size="200"
    level="M"
    render-as="svg"
    :foreground="BARCODE_FOREGROUND"
    :background="BARCODE_BACKGROUND"
  />
</div>
```
- Import: `import QrcodeVue from 'qrcode.vue'`
- `render-as="svg"` is mandatory (SCAN-01 spec).
- Size 200px in detail view; 80px in card tile embed (via `size` prop if BarcodeDisplay is made size-aware, or via a separate small variant — see card tile spec below).
- The outer `div` has `bg-white` — BARCODE_BACKGROUND enforcement.

**Branch B — Linear barcode (barcode_type IN ['code128','ean13','code39'] AND barcode_value truthy):**
```
<div class="bg-white p-6 rounded flex items-center justify-center cursor-pointer"
     @click="emit('scan')">
  <svg ref="barcodeSvgRef"></svg>
</div>
```
Script logic:
```typescript
import JsBarcode from 'jsbarcode'
import { ref, onMounted, watch, useTemplateRef } from 'vue'

const barcodeSvgRef = useTemplateRef<SVGSVGElement>('barcodeSvgRef')
const renderError = ref(false)

function renderBarcode(): void {
  if (!barcodeSvgRef.value) return  // watcher guard — STATE.md BR-5
  try {
    JsBarcode(barcodeSvgRef.value, props.barcode_value ?? '', {
      format: props.barcode_type?.toUpperCase() ?? 'CODE128',
      lineColor: BARCODE_FOREGROUND,
      background: BARCODE_BACKGROUND,
      displayValue: true,
      fontSize: 14,
      margin: 10,
    })
    renderError.value = false
  } catch {
    renderError.value = true  // triggers fallback branch C
  }
}

onMounted(renderBarcode)                          // fires after SVG mounts
watch(() => props.barcode_value, renderBarcode)  // NO immediate:true (STATE.md BR-5)
```
- On catch: `renderError.value = true` — Branch B's error state falls into Branch C rendering (card_number as large text + "Barcode unavailable" caption).
- `format` uses PocketBase select value mapped to JsBarcode format string: `code128` → `'CODE128'`, `ean13` → `'EAN13'`, `code39` → `'CODE39'`.

**Branch C — Barcode unavailable (barcode_value absent OR renderError === true), card_number truthy:**
```
<div class="bg-white p-6 rounded flex flex-col items-center justify-center cursor-pointer gap-2"
     @click="emit('scan')">
  <span class="text-3xl font-bold" style="color: #000000;">
    {{ card_number }}
  </span>
  <span class="text-sm" style="color: var(--color-typo-muted)">Barcode unavailable</span>
</div>
```
- card_number rendered in `text-3xl font-bold` (28px, weight 700) for scan legibility.
- "Barcode unavailable" caption in `text-sm` muted. Never `v-html`.

**Branch D — Neither barcode_value nor card_number (both falsy):**
```
<div class="bg-white p-6 rounded flex flex-col items-center justify-center gap-2">
  <iconify-icon icon="mdi:barcode-off" width="48" height="48"
    style="color: var(--color-typo-muted)" aria-hidden="true"></iconify-icon>
  <span class="text-sm" style="color: var(--color-typo-muted)">No barcode</span>
</div>
```
- No `cursor-pointer` — no scan overlay trigger when there is nothing to scan.
- No `@click` emit.

**Branch selection logic (computed):**
```typescript
type BarcodeDisplayBranch = 'qr' | 'linear' | 'number-fallback' | 'empty'

const displayBranch = computed<BarcodeDisplayBranch>(() => {
  if (props.barcode_type === 'qr' && props.barcode_value) return 'qr'
  if (['code128','ean13','code39'].includes(props.barcode_type ?? '') && props.barcode_value) return 'linear'
  if (props.card_number) return 'number-fallback'
  return 'empty'
})
```
Note: `barcode_type === 'number'` (PocketBase select value) is treated as no-barcode → falls to Branch C/D based on card_number.

**Size note for MembershipCard embed vs MembershipDetail embed:**
BarcodeDisplay renders at full size (200px QR, auto-width linear SVG) in both contexts. In MembershipCard tiles, the barcode is NOT embedded directly — the card tile shows only a barcode-type badge (text label). BarcodeDisplay is embedded only in MembershipDetail and the scan overlay.

### 2. MembershipCard.vue (New component)

**Purpose:** Coloured tile card in the membership grid. Single-click opens MembershipDetail.

**Props:**
```typescript
defineProps<{
  record: Memberships
}>()
```

**Emits:**
```typescript
defineEmits<{
  click: []  // parent opens MembershipDetail
}>()
```

**Tile background binding (locked — STATE.md):**
```typescript
const tileStyle = computed(() => ({
  backgroundColor: props.record.card_color ? '#' + props.record.card_color : '#002244'
}))
```

**Expiry warning computed (locked — STATE.md):**
```typescript
import dayjs from 'dayjs'

const isExpirySoon = computed<boolean>(() => {
  if (!props.record.expiry_date) return false
  const expiryDay = dayjs(props.record.expiry_date)
  const today = dayjs()
  const daysUntil = expiryDay.diff(today, 'day')
  return daysUntil <= 30  // also true when daysUntil < 0 (already expired)
})

const isExpired = computed<boolean>(() => {
  if (!props.record.expiry_date) return false
  return dayjs(props.record.expiry_date).isBefore(dayjs(), 'day')
})

const expiryBadgeText = computed<string>(() => {
  if (!props.record.expiry_date) return ''
  if (isExpired.value) return 'Expired'
  return 'Expiring soon'
})
```

**Template structure:**
```
<Card
  class="cursor-pointer hover:shadow-md transition-shadow overflow-hidden"
  :style="tileStyle"
  style="min-height: 8rem;"
  @click="emit('click')"
>
  <template #content>
    <!-- Card name — primary -->
    <p class="text-lg font-bold mb-1" style="color: #ffffff;">
      {{ record.card_name }}
    </p>

    <!-- Issuer — secondary -->
    <p v-if="record.issuer" class="text-sm mb-2" style="color: rgba(255,255,255,0.75);">
      {{ record.issuer }}
    </p>

    <!-- Barcode type badge -->
    <div class="flex items-center gap-2 mb-2">
      <Badge
        v-if="record.barcode_type && record.barcode_type !== 'number'"
        :value="barcode_type_label"
        severity="secondary"
        class="text-xs"
      />
      <Badge
        v-else-if="record.barcode_type === 'number'"
        value="Number only"
        severity="secondary"
        class="text-xs"
      />
    </div>

    <!-- Expiry date row -->
    <div class="flex items-center gap-2 mt-auto">
      <p v-if="record.expiry_date" class="text-sm" style="color: rgba(255,255,255,0.85);">
        Expires: {{ displayExpiry(record.expiry_date) }}
      </p>
      <!-- Expiry warning badge -->
      <Badge
        v-if="isExpirySoon && record.expiry_date"
        :value="expiryBadgeText"
        :severity="isExpired ? 'danger' : 'warning'"
      />
    </div>
  </template>
</Card>
```

**barcode_type_label computed:**
```typescript
const BARCODE_TYPE_LABELS: Record<string, string> = {
  qr: 'QR',
  code128: 'Code 128',
  ean13: 'EAN-13',
  code39: 'Code 39',
  number: 'Number only',
}

const barcode_type_label = computed<string>(() =>
  BARCODE_TYPE_LABELS[props.record.barcode_type ?? ''] ?? props.record.barcode_type ?? ''
)
```

**Expiry date display:**
```typescript
function displayExpiry(iso: string): string {
  return dayjs(iso).format('DD MMM YYYY')  // compact, e.g. "15 Jan 2027"
}
```

**Hover / focus interaction:**
- `hover:shadow-md transition-shadow` — PrimeVue Card with Tailwind utility classes.
- Keyboard: `tabindex` provided by PrimeVue Card's click handler or use a `<button>` wrapper if PrimeVue Card does not propagate keyboard activation.
- No PrimeVue Card `@keyup.enter` event — wrap with `role="button"` and `tabindex="0"` if needed.

### 4. MembershipDetail.vue (New component)

**Purpose:** Full-field read-only view rendered inside a PrimeVue Dialog. Embeds BarcodeDisplay. Embeds AttachmentPreview for card_image. Contains scan overlay trigger logic.

**Props:**
```typescript
defineProps<{
  record: Memberships
  token: string
}>()
```

**Local state (scan overlay):**
```typescript
const showScanOverlay = ref(false)
const wakeLock = ref<WakeLockSentinel | null>(null)

async function openScanOverlay(): Promise<void> {
  showScanOverlay.value = true
  // Screen wake lock — STATE.md FS-1
  if ('wakeLock' in navigator) {
    try {
      wakeLock.value = await navigator.wakeLock.request('screen')
    } catch {
      // Wake lock denied or not available — degrade silently; overlay still opens
    }
  }
}

async function closeScanOverlay(): Promise<void> {
  showScanOverlay.value = false
  try {
    await wakeLock.value?.release()
  } catch { /* ignore */ }
  wakeLock.value = null
}

// Re-acquire wake lock when tab regains visibility (STATE.md FS-1)
async function onVisibilityChange(): Promise<void> {
  if (document.visibilityState === 'visible' && showScanOverlay.value) {
    if ('wakeLock' in navigator) {
      try {
        wakeLock.value = await navigator.wakeLock.request('screen')
      } catch { /* degrade silently */ }
    }
  }
}

onMounted(() => document.addEventListener('visibilitychange', onVisibilityChange))
onUnmounted(() => document.removeEventListener('visibilitychange', onVisibilityChange))
```

### 5. Scan Overlay (inline in MembershipDetail.vue)

**Overlay structure:** `<Teleport to="body">` wraps the overlay div. This escapes the PrimeVue Dialog's z-index stacking context and ensures `z-index: 9999` is effective even when the Dialog is open beneath.

**Position contract (locked — STATE.md FS-2):**
```css
position: fixed;
inset: 0;             /* top/right/bottom/left: 0 — full viewport */
z-index: 9999;
background: #ffffff;
filter: brightness(1.4);
```

**Brightness filter:** Applied to the entire overlay div. Boosts screen luminance for scanner reflectance without the Fullscreen API. On iOS Safari, `filter: brightness(1.4)` on a fixed full-viewport element is well-supported.

**Wake lock wiring (locked — STATE.md FS-1):**
- `navigator.wakeLock.request('screen')` called inside `openScanOverlay()`.
- Entire call wrapped in `try/catch` — degrade silently if wake lock API unavailable (iOS < 16.4, Firefox Android).
- Re-acquired in `onVisibilityChange` handler when `document.visibilityState === 'visible'` and overlay is open (handles tab-switch and screen-off scenarios).
- Released in `closeScanOverlay()`.

**Close button requirements:**
- `position: absolute; top: 16px; right: 16px` — always visible regardless of barcode height.
- `width: 48px; height: 48px` — minimum touch target.
- Semi-transparent background (`rgba(0,0,0,0.08)`) so button is visible against the white overlay.
- `aria-label="Close scan view"` for screen readers.
- `mdi:close` icon, 24px, color #000000.

**Overlay does NOT have its own BarcodeDisplay `@scan` handler** — the nested BarcodeDisplay inside the overlay is presentational only (no `@scan` emit connected).

## Interaction & Motion

### Card Tile — Hover State

PrimeVue Card with `hover:shadow-md transition-shadow` (Tailwind). Shadow elevates tile visually on hover. `cursor-pointer` set on Card. Transition duration: Tailwind default 150ms ease-in-out.

No background-color change on hover — the card tile's `card_color` background is intentional branding and must not shift on hover.

### Card Tile — Click

1. User taps `MembershipCard` tile.
2. `emit('click')` fires immediately.
3. `MembershipsTab.openDetail(record)` called synchronously setting `selectedRecord`.
4. If `record.card_image` truthy: `pb.files.getToken()` awaited (~200–400ms).
5. `showDetail.value = true` opens Dialog after token resolves.
6. Dialog enters with PrimeVue's default overlay fade-in (~200ms).
7. `MembershipDetail` renders — BarcodeDisplay branch evaluated.

### BarcodeDisplay — Tap to Scan

1. User taps the barcode/QR/card_number display area.
2. `emit('scan')` fires.
3. `MembershipDetail.openScanOverlay()` called.
4. `navigator.wakeLock.request('screen')` attempted (try/catch, non-blocking).
5. `showScanOverlay.value = true`.
6. Teleport overlay renders immediately — no transition animation (instant white fill for scan readiness).
7. BarcodeDisplay re-renders at full overlay width.

### Scan Overlay — Close

1. User taps the close button (top-right, 48px touch target).
2. `closeScanOverlay()` called.
3. `wakeLock.release()` attempted (try/catch).
4. `showScanOverlay.value = false`.
5. Overlay removed from DOM immediately (no exit animation — scan use case prioritises instant dismissal).
6. MembershipDetail Dialog is still open underneath.

### Dialog — Keyboard Navigation

- Escape key: closes Dialog (PrimeVue default). If scan overlay is open, Escape should close the overlay first — this requires a `@keydown.esc` handler on the overlay div calling `closeScanOverlay()` with `event.stopPropagation()`.
- Tab: cycles through focusable elements inside Dialog.
- Focus trap: PrimeVue Dialog traps focus. Scan overlay is rendered via Teleport outside Dialog — it must manage its own focus trap (close button receives focus automatically via `nextTick` after `showScanOverlay = true`).

### Loading State

- `isLoading = true` immediately on `onMounted` entry.
- 3 skeleton cards render in the grid.
- `isLoading = false` in `finally` → grid re-renders with real data or empty state.
- No flash: skeleton replaces in same DOM position.

### Toast Notifications

All toasts via `vue-sonner`'s `toast.error()`. No new Toaster mount needed (exists in App.vue). Position top-right, 4000ms auto-dismiss. Matches VaccinationsTab pattern exactly.
