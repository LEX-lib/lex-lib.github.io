Scan overlay state + wake lock (STATE.md FS-1 + FS-2 locked):
```typescript
const showScanOverlay = ref(false)
const wakeLock = ref<WakeLockSentinel | null>(null)

async function openScanOverlay(): Promise<void> {
  showScanOverlay.value = true
  if ('wakeLock' in navigator) {
    try {
      wakeLock.value = await navigator.wakeLock.request('screen')
    } catch {
      // Degrade silently — overlay still opens without wake lock
    }
  }
}

async function closeScanOverlay(): Promise<void> {
  showScanOverlay.value = false
  try {
    await wakeLock.value?.release()
  } catch { /* ignore */ }
  wakeLock.value = null
}

// Re-acquire when tab regains visibility (STATE.md FS-1 — wake lock released on tab switch)
async function onVisibilityChange(): Promise<void> {
  if (document.visibilityState === 'visible' && showScanOverlay.value) {
    if ('wakeLock' in navigator) {
      try {
        wakeLock.value = await navigator.wakeLock.request('screen')
      } catch { /* degrade silently */ }
    }
  }
}

onMounted(() => document.addEventListener('visibilitychange', onVisibilityChange))
onUnmounted(() => document.removeEventListener('visibilitychange', onVisibilityChange))
```

**Critical implementation notes:**
- `<Teleport to="body">` MUST wrap the overlay div — not the overlay inside any Dialog content slot (Pitfall 3)
- The overlay `<div>` has `@keydown.esc.stop="closeScanOverlay"` — `.stop` prevents the Escape from bubbling to the Dialog and closing it (UI-SPEC §Keyboard Navigation)
- The overlay BarcodeDisplay has NO `@scan` binding — it is presentational only (overlay is already the scan destination)
- `notes` section uses `{{ record.notes }}` mustache — NEVER v-html
- `card_color` swatch uses `:style="{ backgroundColor: '#' + record.card_color }"` — '#' always prepended
- `displayExpiry` is a computed (not a function) — uses `props.record.expiry_date` internally
- Divider is PrimeVue auto-import — no explicit import in script
