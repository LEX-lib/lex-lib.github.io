# Phase 13 Plan 02: ManageMembership.vue Dialog Summary

**One-liner:** ManageMembership.vue create/edit Dialog with direct v-model refs, Zod safeParse, ColorPicker (navy default), conditional barcode_value field, EXIF-stripped FileUpload, and AUTH/pendingFile safety patterns.

## What Was Built

`src/components/projects/wallecx/ManageMembership.vue` — the primary CRUD write surface for membership cards. A PrimeVue Dialog housing a 9-field form that handles both create and edit modes via a null-guarded `defineModel<Memberships | null>("record")` pattern.

**Script setup highlights:**
- 8 direct `ref` bindings for form fields (cardName, barcodeType, barcodeValue, cardNumber, cardColor, expiryDate, issuer, notes) — no `@primevue/forms` per D-07
- `cardColor` initializes to `"002244"` (navy, no `#`) for create mode and as fallback in edit mode (D-01)
- `watch(barcodeType)` clears `barcodeValue` immediately when type switches to `"number"` (D-02)
- `thumbnailUrl` computed returns `pb.files.getURL(...)` only when `isEditMode && record.card_image` (D-03)
- Zod schema with `card_name` min-1 and `card_color` hex-6 regex; `safeParse` on submit with `flatten().fieldErrors` for per-field error refs (Pitfall 7 / Zod 4 API)
- `onFileSelect`: image-only MIME allowlist (`image/jpeg,image/png,image/webp`), EXIF strip via canvas re-encode, `imageCompression` to 1.5 MB / 2048px
- `onSubmit`: D-02 guard on barcode_value append, D-03 guard on card_image append, HIGH-01 null guard on `pb.authStore.record?.id`, D-09/D-10 create vs update branching
- `pendingFile.value = null` in both success branch and `onHide()` (HIGH-02 dual-reset)
- `isSaving` in `finally` block; `isSaving = false` on auth null-guard early return

**Template highlights:**
- `<form @submit.prevent="onSubmit">` — native form, not `<Form zodResolver>` (D-07)
- `v-if="barcodeType !== 'number'"` on barcode_value field (D-02 — DOM removal, not `v-show`)
- ColorPicker with adjacent `<span :style="{ backgroundColor: '#' + cardColor }">` swatch (D-01)
- Edit-mode thumbnail `<img :src="thumbnailUrl">` above FileUpload, helper text "Current image — select a new file to replace it" (D-03)
- `chooseLabel="Replace image"` when `isEditMode && record?.card_image` (D-03)
- All fields `:disabled="isSaving"`; Button `:loading="isSaving" :disabled="isSaving"` (D-08)
