<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Unauthenticated request → PocketBase API | Rules must reject any request without a valid auth token |
| Authenticated user → other users' records | Rules must enforce user = @request.auth.id so User B cannot access User A's membership cards |
| Uploaded card_image URL → browser without token | protected: true forces token requirement; prevents URL-guessing attacks on card photos |
| barcode_type value → PocketBase SELECT field | SELECT field rejects values outside the 5 allowed options server-side; prevents injection of arbitrary barcode format strings |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-01 | Information Disclosure | wallecx_memberships listRule | mitigate | listRule `@request.auth.id != "" && user = @request.auth.id` ensures only the record owner sees their own membership cards; unauthenticated returns 403 |
| T-11-02 | Elevation of Privilege | wallecx_memberships createRule | mitigate | createRule `@request.data.user = @request.auth.id` prevents a user from creating records assigned to another user's id; request rejected if payload user != auth id |
| T-11-03 | Information Disclosure | card_image file field without protected: true | mitigate | protected: true on card_image field requires short-lived token for file access; direct URL without token returns 403; card photos may reveal personal information (club membership, insurance, ID cards) |
| T-11-04 | Tampering | wallecx_memberships updateRule | mitigate | updateRule `user = @request.auth.id` prevents User B from patching User A's membership cards even with a known record id |
| T-11-05 | Tampering | wallecx_memberships deleteRule | mitigate | deleteRule `user = @request.auth.id` prevents User B from deleting User A's membership cards |
| T-11-06 | Tampering | barcode_type as free Text field | mitigate | barcode_type is a SELECT field limiting values to: qr, code128, ean13, code39, number — server-side validation rejects any other value; prevents injection of unrecognised format strings that could crash Phase 12 JsBarcode rendering |
| T-11-07 | Information Disclosure | card_image MIME allowlist including PDF | mitigate | MIME allowlist is image-only (image/jpeg, image/png, image/webp); PDF excluded per D-02; reduces attack surface (no PDF.js processing risk for membership cards) |
| T-11-08 | Information Disclosure | notes field as EditorField | accept | notes is a plain TextField; EditorField (which stores HTML) is explicitly avoided to prevent v-html injection in Phase 12 detail views |
</threat_model>
