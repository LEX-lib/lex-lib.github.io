# Milestone v1.0: Wallecx MVP

**Status:** ✅ SHIPPED 2026-05-12
**Phases:** 0–4
**Total Plans:** 17

## Overview

First release of Wallecx — a personal vaccination records vault built as a Lexarium mini-app at `/projects/wallecx`. Each authenticated user can save and retrieve their own vaccination records including text fields and an attached scan/photo of the card, with server-side per-user privacy enforced via PocketBase collection rules.

## Milestone Summary

**Key Accomplishments:**
1. Stripped VITE_LOGIN_* dev credentials from codebase; added `lint:secrets` grep guard — production bundle no longer carries shipped credentials
2. `wallecx_vaccinations` PocketBase collection with server-side per-user isolation (5 access rules + composite index) — two-user smoke test confirmed cross-user isolation across all access types
3. MIME-branched attachment preview (image `<img>` with thumb / PDF via lazy-loaded `vue-pdf-embed` / download fallback) with short-lived view-time tokens and zero `v-html`
4. Full CRUD write path: Zod-validated PrimeVue dialog, EXIF-stripping image pipeline (canvas re-encode → browser-image-compression), `isSaving` guard preventing double-submits
5. Projects directory tile + design token audit confirming zero bespoke colors; JSON export button with authenticated-user-only data
6. First repo tests: `vaccinationMapper.spec.ts` (mapper field-strip + id-refresh contract) and `guard.spec.ts` (requiresAuth redirect)

**Key Decisions:**
- Mini-app at `/projects/wallecx` — fits Lexarium portfolio pattern; reuses auth, design system, PocketBase
- Server-side per-user isolation as the real auth boundary — PocketBase collection rules, not just route guard
- Single `card` file field for both image and PDF — MIME-sniff at render time
- Two net-new runtime deps only: `browser-image-compression@^2.0.2` and `vue-pdf-embed@^2.1.4`
- `@request.body.user` syntax for createRule (PocketBase v0.29.3+, not the deprecated `@request.data.user`)
- READ-06 (CSP `worker-src`) dropped — pdfjs incompatible with strict meta CSP; security outcome accepted

**Issues Resolved:**
- PocketBase v0.29.3 createRule syntax `@request.body.user` (not `@request.data.user`) — fixed during Phase 1
- Phase 3 code review HIGH-01/02 (null assertion, stale pendingFile) and MEDIUM-03 (date-sorted onCreated) — fixed in Phase 4 plan 01

**Issues Deferred (acknowledged at milestone close 2026-05-13):**
