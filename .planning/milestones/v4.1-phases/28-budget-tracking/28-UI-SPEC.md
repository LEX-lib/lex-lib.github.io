## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline label-to-input spacing |
| sm | 8px | Compact row padding, gap between badge and progress bar |
| md | 16px | Default element spacing, form field vertical gap (`space-y-4`) |
| lg | 24px | Section padding, gap between budget section and chart |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Empty-state vertical padding (`py-12` = 48px top + 48px bottom) |
| 3xl | 64px | Page-level spacing (not used in this phase) |

Exceptions:
- Touch targets: `min-h-[44px]` on all interactive controls (Tab triggers, Buttons, InputNumbers, ToggleButtons) — mandatory Phase 15 invariant.
- Budget progress bar track height: 8px (h-2 in Tailwind = 8px) — below the 4px grid but permissible for decorative track elements.

## Component Inventory

### New Component: ManageBudget.vue

**Layout — Desktop (Dialog):**
- PrimeVue `Dialog`, modal, `:style="{ width: '40vw' }"`, `:breakpoints="{ '960px': '75vw', '641px': '92vw' }"`
- Header text: "Manage Budgets"
- `:closable="!isSaving"`
- Content: scrollable category list + sticky "Save Budgets" button at bottom

**Layout — Mobile (Drawer):**
- PrimeVue `Drawer`, `position="bottom"`, `:show-close-icon="!isSaving"`
- Custom `#header` slot: drag handle (`w-8 h-1 rounded-full bg-gray-300 dark:bg-gray-600`) + "Manage Budgets" title (`font-bold`)
- Content: same form as dialog

**Category row structure (one per category):**
```
[Category name (text-sm, --color-typo-heading)]
[InputNumber (fluid, amount >= 0)] [SelectButton: Monthly | Yearly]
```
- Category name: `text-sm`, color `var(--color-typo-heading)`, no truncation (allow wrap)
- InputNumber: `v-model` typed as `number | null`, `fluid`, `:minFractionDigits="2"`, `:maxFractionDigits="2"`, `:min="0"`, `:disabled="isSaving"` — mirrors ManageExpense.vue
- SelectButton (or ToggleButton pair): "Monthly" / "Yearly" options, bound to `budget_type` per row
- Row gap: `gap-2` between InputNumber and SelectButton
- Row divider: `border-b` using `var(--color-surface-divider)` between each category row
- Pre-population: when dialog opens, seed `amount` and `budget_type` from matching budget record; default `budget_type` = `'monthly'` for rows with no existing record

**Save button:**
- `<Button type="button" fluid label="Save Budgets" :loading="isSaving" :disabled="isSaving" />` at bottom of form
- Triggers upsert loop (create / update / delete per row)

**Post-save:**
- `toast.success('Budgets saved.')` on full success
- `toast.error('Failed to save budgets. Please try again.')` on catch
- Emits `'saved'` event; parent re-fetches budgets ref

**Dark mode scoped overrides (same as ManageExpense.vue):**
```css
:deep(.my-app-dark .p-dialog),
:deep(.my-app-dark .p-dialog .p-dialog-content) {
  background-color: var(--color-surface-card);
  color: var(--color-typo-body);
}
:deep(.my-app-dark .p-drawer),
:deep(.my-app-dark .p-drawer .p-drawer-content) {
  background-color: var(--color-surface-card);
  color: var(--color-typo-body);
}
```

### Extended Component: ExpensesReportsView.vue

**"Manage Budgets" button (D-01):**
- Placement: inside the STATE 4 block (`<template v-else>`), immediately below the Grand Total hero and above the chart. Always visible in STATE 4 regardless of whether any budgets exist.
- Variant: secondary/outlined to avoid visual competition with the grand total hero
- Copy: "Manage Budgets"
- Icon: `pi pi-sliders-h` (left)
- Size: `small`, `min-h-[44px]`
- Alignment: right-aligned (`class="flex justify-end"`) within a `mb-4` wrapper div

**"Budget vs Actual" section (D-04):**
- Rendered with `v-if="visibleBudgets.length > 0"` — entirely absent when period is `this-quarter` or `custom`, or when no matching budgets exist
- Section heading: "Budget vs Actual", `text-base font-bold`, color `var(--color-typo-heading)`, `mt-6 mb-3`
- One row per category that has a budget for the active period type

**Budget row structure:**
```
[Category name]
[Progress bar track]  [Actual amount]  [Under/Over badge]
```
- Category name: `text-sm`, color `var(--color-typo-heading)`, `mb-1`
- Progress bar track: `w-full`, `h-2` (8px), `rounded-full`, background `var(--color-surface-divider)`
- Progress bar fill: `h-2 rounded-full transition-all`, width = `Math.min(100, (actual / budget) * 100) + '%'`, color = `var(--color-status-success)` when actual <= budget, `var(--color-status-error)` when actual > budget
- Actual amount row: `flex items-center justify-between mt-1`
  - Left: `formatCurrency(actual)`, `text-xs`, color `var(--color-typo-muted)`
  - Right: badge (see below)
- Badge: `text-xs font-normal px-2 py-0.5 rounded-full`
  - "Under by X": background tint of success color; text copy `Under by {{ formatCurrency(budget - actual) }}`; `style="background-color: color-mix(in srgb, var(--color-status-success) 15%, transparent); color: var(--color-status-success)"`
  - "Over by X": background tint of error color; text copy `Over by {{ formatCurrency(actual - budget) }}`; `style="background-color: color-mix(in srgb, var(--color-status-error) 15%, transparent); color: var(--color-status-error)"`
  - Exactly on budget: badge text "On budget", success color treatment
- Row spacing: `mb-4` between budget rows
- Row `aria-label`: `"Budget for [category]: [budget amount] budget, [actual amount] actual, [under/over]"`

## Interaction Contracts

### ManageBudget Dialog/Drawer

| Trigger | Behavior |
|---------|----------|
| "Manage Budgets" button click | Sets `showManageBudget = true`; opens Dialog (desktop) or Drawer (mobile) |
| Dialog/Drawer opens | `watch(visible)` fires; local rows array seeded from `:budgets` prop — pre-fills amount and budget_type per category |
| User edits InputNumber | Updates local row state only; no immediate PocketBase call |
| User toggles Monthly/Yearly | Updates `budget_type` on local row state only |
| "Save Budgets" button click | Triggers upsert loop; `isSaving = true`; button shows loading spinner; closable = false |
| Save succeeds | `toast.success('Budgets saved.')`; emits `'saved'`; `visible = false` |
| Save fails (any error) | `toast.error('Failed to save budgets. Please try again.')`; dialog stays open; `isSaving = false` |
| User clears an amount to 0 or blank | Treated as "no budget" on save — deletes existing record or skips |
| Dialog closed while saving | Not possible — `:closable="!isSaving"` and `:show-close-icon="!isSaving"` prevent close during save |

### Budget vs Actual Section

| Condition | Behavior |
|-----------|----------|
| `period === 'this-month'` | Show only budgets with `budget_type === 'monthly'`; section visible if any exist |
| `period === 'this-year'` | Show only budgets with `budget_type === 'yearly'`; section visible if any exist |
| `period === 'this-quarter'` | Section hidden entirely (`v-if` = false) |
| `period === 'custom'` | Section hidden entirely (`v-if` = false) |
| No categories have matching budgets | Section hidden entirely (no "empty budgets" placeholder) |
| Actual > budget | Progress bar fills to 100%; fills with `--color-status-error`; badge shows "Over by $X" |
| Actual <= budget | Progress bar fills proportionally; fills with `--color-status-success`; badge shows "Under by $X" or "On budget" |
| Parent emits `'saved'` | ExpensesTab.vue re-fetches `wallecx_expense_budgets` collection; `budgets` ref updated; ExpensesReportsView re-renders section |

## Accessibility Contract

- All interactive controls maintain `min-h-[44px]` touch target (Phase 15 invariant).
- InputNumber fields have a visible label above them matching the category name.
- Progress bar rows use `aria-label` with category name, budget amount, actual amount, and under/over status.
- Drawer drag handle is `aria-hidden="true"` (decorative).
- Badge color is not the only indicator — text ("Under by" / "Over by") conveys status independently of color.
- `toast.success` / `toast.error` from vue-sonner are announced by screen readers (existing infrastructure).
- Dialog is modal with `modal` prop — background not interactive while open.
