**`<script setup lang="ts">`:**

```typescript
import { defineAsyncComponent, computed, ref } from "vue";
import { pb } from "@/lib/pocketbase";
import type { Vaccinations } from "@/types/wallecx/vaccinations/types";

const VuePdfEmbed = defineAsyncComponent(() => import("vue-pdf-embed"));

const props = defineProps<{
  record: Vaccinations;
  token: string;
}>();

const pdfFailed = ref(false);

function getMimeCategory(filename: string): "image" | "pdf" | "unknown" {
  const lower = filename.toLowerCase();
  if (
    lower.endsWith(".jpg") ||
    lower.endsWith(".jpeg") ||
    lower.endsWith(".png") ||
    lower.endsWith(".webp")
  )
    return "image";
  if (lower.endsWith(".pdf")) return "pdf";
  return "unknown";
}

const mimeCategory = computed(() => getMimeCategory(props.record.card));

const tokenUrl = computed(() =>
  pb.files.getURL(props.record, props.record.card, { token: props.token })
);

const thumbUrl = computed(() =>
  pb.files.getURL(props.record, props.record.card, {
    thumb: "400x400",
    token: props.token,
  })
);

function onPdfError(): void {
  pdfFailed.value = true;
}
```

**`<template>`:**

Root guard: `v-if="record.card"` prevents calling `pb.files.getURL()` with an empty filename (Pitfall 5). Separate `v-else` renders the no-attachment fallback.

Three branches inside the `v-if`:

**Image branch** (`mimeCategory === 'image'`):
```html
<img
```

**PDF branch** (`mimeCategory === 'pdf'`) — Suspense required:
```html
<Suspense v-else-if="mimeCategory === 'pdf'">
  <VuePdfEmbed
    :source="tokenUrl"
    :page="1"
    class="w-full"
    @loading-failed="onPdfError"
  />
  <template #fallback>
    <div class="flex flex-col items-center py-6 gap-2">
      <Skeleton height="12rem" class="w-full" />
      <p class="text-sm" style="color: var(--color-typo-muted)">Loading PDF preview…</p>
    </div>
  </template>
</Suspense>
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| PocketBase API → Browser | Signed file URLs with short-lived tokens flow from PocketBase to the browser; token is appended as a query param |
| User file content → PDF renderer | PDF files stored in PocketBase are rendered by pdfjs-dist in a Web Worker; malicious PDFs could exploit pdfjs-dist CVEs |
| CSP → Browser enforcement | The CSP meta tag is enforced by the browser; any relaxation broadens the attack surface |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01-01 | Tampering | `AttachmentPreview.vue` — notes field | mitigate | `notes` rendered with `{{ }}` mustache only — no `v-html`; verified by `git grep "v-html" src/components/projects/wallecx` returning zero hits |
| T-02-01-02 | Information Disclosure | `pb.files.getURL` — unprotected file URL | mitigate | All file URL calls include `{ token: props.token }` option; token is short-lived (~2 min) and generated at view time (READ-07); requests without token return 403 from PocketBase |
| T-02-01-03 | Elevation of Privilege | `vue-pdf-embed` / `pdfjs-dist` — PDF.js CVE-2024-4367 | mitigate | `pdfjs-dist@4.10.38` is pinned in package-lock.json (exceeds the 4.2.67 minimum that patches CVE-2024-4367); `worker-src 'self' blob:` narrows worker execution to same-origin blobs only |
| T-02-01-04 | Elevation of Privilege | CSP meta tag — script-src broadening | mitigate | Only `worker-src 'self' blob:` is appended; `script-src` remains `'self'`; verified by grep check that `script-src` does not contain `blob:` |
