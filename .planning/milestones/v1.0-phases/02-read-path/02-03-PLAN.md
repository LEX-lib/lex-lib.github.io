**`<script setup lang="ts">`:**

```typescript
import dayjs from "dayjs";
import type { Vaccinations } from "@/types/wallecx/vaccinations/types";

const props = defineProps<{
  record: Vaccinations;
  token: string;
}>();

function displayDate(iso: string): string {
  return dayjs(iso).format("DD MMMM YYYY");
}
```

Note: `AttachmentPreview` is auto-resolved by unplugin-vue-components (same as PrimeVue components) once the file exists in `src/components/projects/wallecx/`. No explicit import needed.

**CRITICAL constraints:**
- NEVER use `v-html` for any field — all user-supplied strings use `{{ }}` mustache only (READ-02)
- Date in detail view uses `"DD MMMM YYYY"` format (e.g. "15 January 2025") — more verbose than list's "DD MMM YYYY"
- `dose_number` uses `??` (nullish coalescing) because it is `number | undefined` — `||` would also treat `0` as falsy
- `lot_number`, `manufacturer`, `location` use `||` (logical OR) because they are strings that may be empty
- Notes section uses `v-if="record.notes"` — omit the section entirely when falsy (don't render an empty block)
- Do NOT add Edit or Delete buttons — Phase 2 is strictly read-only (D-02)
- Do NOT include a `<Dialog>` wrapper — the Dialog is in WallecxApp.vue; this component is the body content only
