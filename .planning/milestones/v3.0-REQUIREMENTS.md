# Requirements: Lexarium v3.0 Site-Wide Dark Mode (Archive)

**Defined:** 2026-05-18
**Shipped:** 2026-05-19
**Core Value:** Every Lexarium surface renders correctly in both light and dark themes; users can toggle between them or rely on OS preference. Marks the shift from "individual themed mini-apps" to "themed platform."

## v3.0 Requirements (Shipped 13/13)

### Theme Infrastructure

- [x] **THEME-01**: `useTheme` composable in `src/composables/useTheme.ts` exposes reactive `Ref<'light' | 'dark'>` state. On first call, reads `localStorage.getItem('lexarium:theme')` — if present, uses that value; otherwise reads `window.matchMedia('(prefers-color-scheme: dark)').matches` to determine OS preference. (Phase 19)
- [x] **THEME-02**: When the theme value changes, the composable applies/removes the `.my-app-dark` class on `document.documentElement` (the `<html>` element), and writes the current value to `localStorage` under the key `lexarium:theme`. (Phase 19; inline FOUC script in `index.html` ensures no flash on hard reload)
- [x] **THEME-03**: NavBar in `CustomNavBar.vue` shows a sun/moon toggle button using `<iconify-icon>` (mdi:weather-sunny / mdi:weather-night) with swapping `aria-label`. Clicking flips the theme; state persists across reloads and navigation. (Phase 19)

### Site Shell & Non-App Pages

- [x] **THEME-04**: HomeView, HeroSection, and AboutMeSection render correctly in dark mode. (Phase 20; `--color-mix-target` CSS variable added to base.css; navy `rgba()` decorative blobs swapped to amber via `:global(.my-app-dark)` overrides; existing `@theme` tokens auto-switched via Phase 18 base.css block)
- [x] **THEME-05**: ProjectsView renders correctly in dark mode. (Phase 20; WIP/Active chip tags re-tuned with legible text colors + `!important` matching)
- [x] **THEME-06**: BlogView renders correctly in dark mode. (Phase 20; `dark:opacity-80` on branding logo)
- [x] **THEME-07**: Login screen renders correctly in dark mode. (Phase 20; `dark:text-white*` paired on 3 hardcoded `text-black*` classes — planner caught Login uses `text-black` not `text-gray-*` as initial scout assumed)
- [x] **THEME-08**: CustomNavBar renders correctly in dark mode. (Phase 20; `dark:invert` on brand-mark Avatar)

### Mini-App Dark Mode Sweep

- [x] **THEME-09**: LexTrack renders correctly in dark mode. (Phase 21-01; **architectural deviation: Option A semantic-token refactor** — file was dark-only-hardcoded, mechanical sweep didn't apply; refactored to `bg-surface-page`/`bg-surface-card`/`text-typo-heading`/etc. so LexTrack is now genuinely theme-aware in BOTH themes — fixes light mode that never worked previously)
- [x] **THEME-10**: Larga renders correctly in dark mode. (Phase 21-02; `:global(.my-app-dark)` override on Leaflet geocoder wrapper + input + dropdown alternatives + hover state; map tiles intentionally stay light in both themes per D-07)
- [x] **THEME-11**: MonitoX (Gift Exchange) renders correctly in dark mode. (Phase 21-03; mechanical sweep ~123 utility pairings across 4 sub-routes; 9 colored-chip `/30` opacity pairings; status colors theme-independent per D-13. UAT follow-up: 6 `bg-black text-white` buttons inverted to `dark:bg-white dark:text-black` for high-contrast on dark page)
- [x] **THEME-12**: API Playground renders correctly in dark mode. (Phase 21-04; dark-by-design Postman-style navy+amber palette confirmed; 13 chrome `.my-app-dark` legibility tweaks added; no third-party syntax highlighter swap needed — uses custom inline regex)

### Wallecx Audit

- [x] **THEME-13**: Wallecx renders correctly in dark mode when toggled via NavBar. (Phase 22; verification-only audit, zero source changes; all 6 regression vectors PASS — toggle interaction, route transitions, FOUC, bottom sheets, BR-2 invariant preserved, PWA standalone mode verified on Vercel preview)

## Architectural Deviations Logged

1. **Phase 21-01 LexTrack — Option A semantic-token refactor** (user-approved at checkpoint). Original plan assumed mechanical `dark:` pairing; reality was the file was dark-only hardcoded (`bg-gray-900`, `text-white`, etc. throughout). Refactored to semantic Phase-18 tokens. Side-benefit: LexTrack now works in light mode for the first time.
2. **Phase 21-04 API Playground — chrome-only overrides** (executor-determined). File was dark-by-design (Postman-style palette); 184 color references but mostly intentional. Applied 13 targeted legibility tweaks rather than a full sweep.

## Out of Scope (decisions held)

| Feature | Reason |
|---------|--------|
| Per-user theme synced via PocketBase | Schema migration + auth-gated logic; localStorage is sufficient for v3.0 |
| Smooth animated theme transitions | Polish item; instant toggle is acceptable |
| Additional theme variants (sepia, high-contrast) | Out of scope for v3.0 |
| In-app theme picker / customisation | Light/dark only |
| Auto-detect time-of-day theme switching | OS-level concern |
| Leaflet dark tile provider | Map tiles intentionally stay light |
| Semantic-token refactor of MonitoX | Cleanup; defer to a future polish phase |
