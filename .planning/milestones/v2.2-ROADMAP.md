# Milestone v2.2: Sort and Search for Membership Cards

**Status:** ✅ SHIPPED 2026-05-15
**Phases:** 16
**Total Plans:** 2

## Overview

The membership cards tab gained a persistent toolbar — mirroring the vaccination toolbar introduced in Phase 7. WallecxToolbar was decoupled from vaccination-specific sort labels (sortOptions promoted to a required prop), and MembershipsTab was wired with real-time search by card name or issuer, four-mode sort, sessionStorage persistence for the selected sort mode, and an informative no-results empty state. All filtering and sorting is pure client-side computed on the already-loaded memberships list.

## Milestone Summary

### Key Accomplishments

1. **WallecxToolbar decoupled** — sortOptions promoted from a hardcoded internal const to a required TypeScript prop; hardcoded vaccination sort labels removed from the generic toolbar; TypeScript enforces every caller must supply their own sort options array
2. **VaccinationsTab owns its sort config** — `vaccinationSortOptions` declared in VaccinationsTab and passed via `:sort-options` prop; existing sort behavior preserved exactly with zero regression
3. **MembershipsTab real-time search** — `displayedMemberships` computed filters the loaded `records` ref by `card_name`/`issuer` (case-insensitive partial match); WallecxToolbar wired with `v-model:search-query` binding
4. **Four-mode sort for membership cards** — Name A–Z, Issuer A–Z, Expiry Date (soonest first; no-expiry cards sorted last), Recently Added; selected mode persisted to `sessionStorage` and restored on mount with whitelist validation
5. **No-results empty state** — informative message with `mdi:magnify-remove-outline` icon and "Clear search" button when `displayedMemberships.length === 0 && searchQuery`

### Key Decisions

- **sortOptions is required with no withDefaults default** — intentional; prevents silent fallback to vaccination-specific sort labels when reusing the toolbar in a different tab
- **displayedMemberships as computed, not function** — Vue tracks reactive dependencies correctly; `searchQuery` and `sortMode` refs both trigger re-evaluation automatically
- **v-model:view-mode bound to string literal `'grid'`** — memberships tab has no view toggle (`show-toggle="false"`); no `viewMode` ref required
- **sessionStorage whitelist validation** — stored sort value is validated against `membershipSortOptions.map(o => o.value)` before assignment; unrecognised values fall back to `'recently-added'`
- **No-results condition uses `&& searchQuery`** — ensures the empty-search state only activates during an active search, not when `records` is genuinely empty on first load

### Issues Resolved

- WallecxToolbar was tightly coupled to vaccination sort labels — refactored to accept any `{ value: string; label: string }[]` array via required prop
- `v-model` on a string literal caused a Vue warn in the template — replaced with one-way `:view-mode="'grid'"` bind
