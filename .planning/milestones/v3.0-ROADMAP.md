## Lessons / Patterns to Carry Forward

- **Mechanical-sweep precondition check** — before executing a mechanical-pairing plan, executor should `grep` to confirm the file actually has light-default utilities; if it's dark-hardcoded, surface a checkpoint
- **PrimeVue CSS specificity** — `!important` is often legitimate for component-library theme overrides because the library's rules are injected at runtime at equal specificity AFTER our scoped CSS
- **PWA UAT requires Vercel preview** — `npm run dev` doesn't generate the manifest or register the SW; localhost testing for PWA install flow is impossible
- **Custom @theme tokens are theme-aware when defined in both blocks** — light values in `@theme {}`, dark values in `.my-app-dark {}` — Tailwind utilities then auto-switch with the class
- **Consolidated HUMAN-UAT.md per phase** — works better than per-plan UAT docs; one walk through, fewer sign-off blocks to track
