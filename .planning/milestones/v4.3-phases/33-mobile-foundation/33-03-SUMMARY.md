# Phase 33 Plan 03: ANALYZE-Gated Bundle Visualizer Summary

Wired `rollup-plugin-visualizer` (+ `cross-env`) into the build behind a strict `process.env.ANALYZE === "true"` gate and added an `npm run analyze` script — producing an on-demand `dist/stats.html` treemap for Phase 36 chunk-split sequencing without affecting the normal build pipeline (FND-03).

## What Shipped

- **`package.json`** — `rollup-plugin-visualizer@^7.0.1` and `cross-env@^10.1.0` added to `devDependencies`; `"analyze": "cross-env ANALYZE=true vite build"` script added immediately after `build-only` for discoverability.
- **`vite.config.ts`** — `import { visualizer } from "rollup-plugin-visualizer"` at the top; the visualizer plugin appended to the `plugins` array via the same spread-conditional idiom already used for `vueDevTools`, gated on `process.env.ANALYZE === "true"`. Config: `filename: "dist/stats.html"`, `template: "treemap"`, `gzipSize: true`, `brotliSize: true`.

## Chosen Report Path & Gating Idiom

- **Report path:** `dist/stats.html` — lives inside `dist/`, which is already git-ignored (`.gitignore` line 4). The artifact is regenerated on demand by `npm run analyze` and is never committed. No `.gitignore` change was required.
- **Gating idiom:** `...(process.env.ANALYZE === "true" ? [visualizer({ ... })] : [])`. This mirrors the existing `...(process.env.NODE_ENV !== "production" ? [vueDevTools()] : [])` line, so the visualizer attaches ONLY when `cross-env` sets `ANALYZE=true` (via `npm run analyze`).
