<objective>
Convert WallecxApp.vue from eager-loading 3 tab components to lazy-loading via `defineAsyncComponent`, with `<Suspense>` boundaries INSIDE each `<TabPanel>` that fall back to layout-matched `<WallecxSkeleton>` instances. This is the largest single contributor to the ≥50% initial-Wallecx-chunk reduction target (D-36-03) because chart.js (via ExpensesReportsView in ExpensesTab), JsBarcode (via BarcodeDisplay in MembershipsTab), and browser-image-compression (via Manage* dialogs in all 3 tabs) all stop landing in the route chunk.

Purpose: Each tab's heavy dependencies download only when the user actually selects that tab. The Suspense fallback uses the same shared skeleton consumers will use in Wave 2/Plan 36-03..05, so the cold-load → loaded transition is CLS-clean (≤0.1).

Output: A single modified file (WallecxApp.vue). Parallel-safe with Plans 36-03/04/05 (file-disjoint).
</objective>
