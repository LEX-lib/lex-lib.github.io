## Touch-Target Contract (LT-01 / D-34-02)

This section is specific to Phase 34's audit mandate. It is the primary visual contract for the executor.

### Global CSS Rules (append to `src/assets/wallecx-overrides.css`)

**Rule 1 — Icon-only PrimeVue buttons:**
```css
/* Phase 34 LT-01: 44×44 touch-target floor for icon-only PrimeVue buttons.
 * Root cause: main.ts applies p-button-sm globally via pt.button.root, and
 * Aura's button.sm.iconOnlyWidth = 2rem (32px) — below the 44px floor.
 * .p-button-icon-only is added by PrimeVue only when NO label text is present.
 * Label buttons are unaffected (height is set by padding + line-height).
 * No !important needed — specificity is sufficient over Aura element-level rules. */
.p-button.p-button-icon-only {
  min-width: 44px;
  min-height: 44px;
}
```

**Rule 2 — WallecxApp top-level Tab triggers:**
```css
/* Phase 34 LT-01: Top-level WallecxApp tab triggers only.
 * Requires class="wallecx-main-tabs" on <Tabs> in WallecxApp.vue.
 * Period tabs (.wallecx-period-tabs) and sub-tabs (.wallecx-sub-tabs) are
 * excluded — they manage their own min-height via scoped :deep rules. */
.wallecx-main-tabs .p-tab {
  min-height: 44px;
}
```

**Rule 3 — Sticky TabList + sticky toolbar (mobile only):**
```css
/* Phase 34 LT-05/D-34-01: Sticky TabList on mobile (≤639px).
 * Overrides PrimeVue's .p-tablist default (position:relative; overflow:hidden).
 * The page (not the Card) is the scroll container for WallecxApp on mobile.
 * overflow:visible removes the clip so the active ink-bar renders correctly.
 * !important required — PrimeVue base CSS sets position:relative at equal specificity. */
@media (max-width: 639px) {
  .wallecx-main-tabs .p-tablist {
    position: sticky !important;
    top: env(safe-area-inset-top);
    z-index: 10;
    background: var(--p-tabs-tablist-background);
    overflow: visible !important;
  }

  /* Phase 34 LT-05: Per-tab sticky toolbar. Class added by tab components via
   * :class="isMobile ? 'wallecx-tab-toolbar' : ''". Top offsets below TabList
   * (≈44px tab height + env(safe-area-inset-top) for notch clearance).
   * border-bottom provides visual separation from scrolling content below. */
  .wallecx-tab-toolbar {
    position: sticky;
    top: calc(env(safe-area-inset-top) + 52px);
    z-index: 9;
    background: var(--p-card-background, white);
    border-bottom: 1px solid var(--p-card-border-color, #e5e7eb);
  }

  /* Dark-mode toolbar background — matches existing .my-app-dark .p-card override
   * (--p-card-background: #1a1f2e set in wallecx-overrides.css line 53). */
  .my-app-dark .wallecx-tab-toolbar {
    background: #1a1f2e;
  }
}
```

**Rule 4 — Bottom-Drawer safe-area insets:**
```css
/* Phase 34 LT-03/D-34-04: Safe-area bottom padding inside bottom Drawers.
 * Pads .p-drawer-content so the last form field / action button clears the
 * iOS home indicator / Android gesture bar.
 * Uses padding-bottom — does NOT change the 85dvh panel height.
 * max() ensures at least 1rem even on devices with no home indicator. */
.p-drawer-bottom .p-drawer-content {
  padding-bottom: max(env(safe-area-inset-bottom), 1rem);
  padding-left: env(safe-area-inset-left);
  padding-right: env(safe-area-inset-right);
}
```

### Scan Overlay Safe-Area Detail

The scan overlay in `MembershipDetail.vue` uses `position:fixed; inset:0; z-index:9999; background:#ffffff`. On a notched iPhone, the notch overlaps the centered barcode content.

**Required changes to the overlay `<div>`:**
```html
style="z-index: 9999; background: #ffffff; filter: brightness(1.4);
       padding-top: env(safe-area-inset-top);
       padding-bottom: env(safe-area-inset-bottom);"
```

**Required change to the close button (currently `class="absolute top-4 right-4"`):**
- Remove Tailwind `top-4`
- Add inline `style="top: calc(1rem + env(safe-area-inset-top)); background: rgba(0,0,0,0.08);"`
- The close button must remain 48×48px (`w-12 h-12`) — already compliant with the 44px floor

No JavaScript, no composable import needed for the overlay — direct `env()` strings in Vue inline style attributes are resolved natively by the browser.

**BR-2 safety note:** These padding changes affect layout around the barcode centring only. They do not touch `BarcodeDisplay.vue`, the `BARCODE_FOREGROUND`/`BARCODE_BACKGROUND` constants, or the `bg-white` class. ## Sticky Chrome Contract (LT-05 / D-34-01)

### What Sticks (mobile ≤639px ONLY)

1. **Layer 1 — TabList:** `position: sticky; top: env(safe-area-inset-top); z-index: 10`
   - Applied via CSS-only override on `.wallecx-main-tabs .p-tablist` at `@media (max-width: 639px)`
   - No JavaScript. No template structural change to WallecxApp.vue beyond adding `class="wallecx-main-tabs"` to `<Tabs>`
   - Background: `var(--p-tabs-tablist-background)` — prevents content bleed-through on scroll

2. **Layer 2 — Per-tab toolbar:** `position: sticky; top: calc(env(safe-area-inset-top) + 52px); z-index: 9`
   - Applied by CSS class `.wallecx-tab-toolbar` triggered from each tab component: `:class="isMobile ? 'wallecx-tab-toolbar' : ''"`
   - 52px approximates TabList height (44px `min-height` + ~8px Aura padding). This is an accepted approximation per RESEARCH.md §Assumptions Log A3. If post-implementation visual check shows the toolbar offsets incorrectly, adjust the constant.
   - Background: `var(--p-card-background, white)` (light) / `#1a1f2e` (dark, `.my-app-dark`)
   - Visual separator: `border-bottom: 1px solid var(--p-card-border-color, #e5e7eb)`

3. **Desktop (≥640px):** Both layers render at `position: static` (default). No sticky behavior on tablet or desktop.

### iOS URL-Bar Jitter Safety

No `100vh` or `h-screen` exists anywhere in `src/components/projects/wallecx/` (verified grep: 0 matches, NFR-DVH-NOT-VH clean). `position: sticky` with `dvh`-based heights does not cause iOS URL-bar collapse jitter. This is the reason D-34-05 confirms the invariant rather than performing a migration.

### Ink-Bar Safety

PrimeVue's `.p-tablist-active-bar` is `position: absolute; inset-block-end: 0` inside `.p-tablist`. Removing `overflow: hidden` on `.p-tablist` (required for sticky to work correctly) could theoretically cause the ink bar to render outside the tablist box. If this occurs during the human-verify checkpoint, apply `clip-path: inset(0)` on `.wallecx-main-tabs .p-tablist` as a fallback — this clips visual overflow without affecting sticky positioning.

### Overflow Handling Override

PrimeVue sets `.p-tablist { position: relative; overflow: hidden }` in its base CSS at equal specificity. The overrides rule uses `!important` on both `position` and `overflow` to win the source-order battle:

```css
position: sticky !important;
overflow: visible !important;
```
